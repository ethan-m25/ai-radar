GPT-5.6 家族正式登场，开源继续追赶 —— 新模型不一定更好

本周 11 条新信号里出现了 2 个 🚨 —— GPT-5.6 Sol Ultra 进入 Codex（7/6）和 腾讯 Hy3 295B 开源模型冲击 GLM-5.2（7/7）。随后 GPT-5.6 完整系列正式发布（7/9）、Grok 4.5 入场（7/9）、Sol Ultra 证明数学猜想（7/11）。但与此同时，“模型越强工具越差”的讨论持续发酵，Sonnet 5、Opus 4.8 甚至 GPT-5.5 在第三方工具调用上出现退化。本周是选择增多但决策变难的一周。

本周一句话总结：GPT-5.6 的完整发布和开源模型的持续追赶，让编码 agent 市场进入了真正的“性价比战争”——你有比以往更多的选择，但每个选择都有隐藏成本（tokenizer 差异、工具兼容性、缓存折扣等）。如果你还没规划多模型策略，现在是时候了。

🏆 本周最重要的唯一一件事

GPT-5.6 正式发布与多模型竞争格局的成形 —— 你的编码工具获得了一次根本性升级

本周的核心事件链是 GPT-5.6 家族从预览到全面可用。周一（7/6）Sol Ultra 进入 Codex，你每天用的工具突然获得了 OpenAI 最强的旗舰模型。周四（7/9）OpenAI 正式发布完整三档定价：Sol $5/$25 每百万 token、Terra $2/$10、Luna $1/$5，并配有 90% 缓存读取折扣。Sam Altman 称 Sol 是“我们生产过的最好的模型”，Simon Willison 的评测显示它在编码 agent 上得分接近 Fable 5，但成本只有三分之一。

但与此同时，腾讯 Hy3 开源（295B MoE，Apache 2.0）在多个基准上紧逼 GLM-5.2（7/7），xAI 发布 Grok 4.5（Cursor 联合训练，$2/$6 定价，7/9）。竞争者越来越多，且定价越来越低。更值得注意的是 Armin Ronacher 和 Simon Willison 独立发现的“模型越强工具越差”现象：Opus 4.8、Sonnet 5、GPT-5.5 在调用第三方工具时凭空发明字段，导致调用失败。原因是 RL 训练过度优化了特定工具（如 Claude Code 内置编辑器），牺牲了对开放生态的兼容性。

如果本周只做一件事：登录 Codex，确认 Sol Ultra 是否已可用。然后创建一个简单的测试脚本，对比 Sol 和之前使用的模型（Sonnet 5 或 Opus 4）在处理你日常任务（数据转换、文件重构、shell 命令生成）时的输出质量和响应速度。同时注意：如果你使用非官方 agent 框架（Pi、Continue 等），先锁定旧模型版本，等待 Anthropic/OpenAI 修复退化问题。

📈 正在成形的趋势

三条在本周被确认或加速的方向

① 模型选择的经济学——性价比取代绝对性能成为首要决策因素

GPT-5.6 的三档定价（Luna $1/$5, Terra $2/$10, Sol $5/$25）加上 90% 缓存折扣，使成本管理变成一项新技能。相比之下 Opus 4.8 是 $15/$75。如果你的日用量是 10M 输入 + 2M 输出，Sonnet 5 每天约 $40，Sol 约 $100，Luna 仅 $20。同时 Hy3 在 OpenRouter 上免费到 7 月 21 日，Grok 4.5 比 Opus 便宜 10 倍。这意味着你可以根据任务复杂度动态切换模型，而不用绑定单一供应商。Simon Willison 的成本案例（用 Fable 5 花 $149 写完 sqlite-utils 4.0rc2）说明预算可以大幅压缩。

② 开源模型持续接近闭源——编码 agent 的去中心化选择增多

腾讯 Hy3（295B MoE, 21B 活跃）在编码和 agent 基准上紧逼 GLM-5.2，且 FP8 只需 ~60GB（单卡 A100 可跑）。加上之前 Qwen 3.6 27B 被验证为本地开发甜点、Ornith-1.0 自改进 RL、Leanstral 1.5 完整框架，开源编码模型的生态已经具备“替换闭源”的潜力。Alibaba 封禁 Claude Code（上周）和 Claude Code 隐写标记争议（上周）进一步推动了企业对开源替代品的兴趣。Copilot 已支持 Kimi K2.7，ZCode 给了 GLM-5.2 官方 IDE。开源不再只是“备选”，而是正在成为主流选项。

③ RL 训练的副作用——新模型在开放生态中可能倒退

Armin Ronacher 的《Better Models: Worse Tools》揭示了结构性问题：Anthropic 用 RL 训练新模型更擅长 Claude Code 的编辑工具，但导致模型在调用第三方工具时发明不存在字段。Simon Willison 独立确认了这个问题，社区补充称 GPT-5.5 和 Gemini 3.5 Flash 也有类似行为。这提醒我们“升级 = 更好”的假设不再成立。如果你依赖工具调用的精确性（比如用 Pi、Continue、自定义 agent），建议锁定旧模型（Sonnet 4.5 / Opus 4.5），直到模型提供商修复兼容性问题。
