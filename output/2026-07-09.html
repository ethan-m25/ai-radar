Grok 4.5 发布 — Cursor 联合训练，1.5T 参数，$2/$6 定价

clawii，xAI 今天正式发布了 Grok 4.5，一个专注于编码和 agent 的旗舰模型。Elon Musk 说它是 "Opus 级别" 但更快、更省 token、更便宜。最值得注意的是 Cursor 参与了联合训练，并称这是他们最强大的模型。定价 $2/$6（输入/输出每百万 token），比 Opus 4.8 便宜不少。

📍 背景 · xAI 之前的主力模型是 Grok 4.3（约 500B 参数）。Grok 4.5 直接跳到 1.5T 参数（3 倍），采用 MoE 架构，上下文窗口 1M token（即将恢复）。定价策略明显走性价比路线：$2/M 输入、$6/M 输出，缓存命中还有折扣。Cursor 的参与意味着这个模型在编码 agent 场景下经过了针对性优化。早期生态支持包括 Grok Build/API、Hermes Agent、Portal 和 OpenRouter。

为什么这事重要

  • 编码 agent 又多了一个选择 — Cursor 联合训练意味着 Grok 4.5 在编码场景下可能表现优秀。如果你用 Cursor，很快就能试到。
  • 定价很有竞争力 — $2/$6 比 Opus 4.8（$15/$75）和 Sonnet 5（$2/$10）都便宜。如果性能接近，性价比优势明显。
  • 1.5T 参数是当前最大公开模型之一 — 虽然 MoE 架构下实际激活参数少很多，但这个规模本身说明 xAI 在认真追赶。

术语解码 · MoE (Mixture of Experts) = 混合专家模型，一种架构，模型由多个"专家"子网络组成，每次只激活一部分，在保持大参数量的同时降低计算成本。Token = 模型处理的最小单位，大约 1 个英文单词 ≈ 1-2 个 token，1 个中文字 ≈ 2-3 个 token。

Bun 从 Zig 重写到 Rust — 编码 agent 的工程极限案例

clawii，Simon Willison 今天详细记录了 Bun 用 Mythos/Fable 从 Zig 重写到 Rust 的工程案例。这不是一个新工具，而是一个展示编码 agent 在大型项目中真实能力的案例研究。Jarred Sumner 用 Claude 的 Mythos/Fable 模型在 11 天内完成了 100 万行代码的重写，成本约 $165,000。

📍 背景 · Bun 是一个 JavaScript/TypeScript 运行时，最初用 Zig 编写。Jarred Sumner 决定用 Rust 重写，主要原因是 Zig 的手动内存管理导致大量 use-after-free 和 double-free bug。重写过程中，Claude 的 Mythos/Fable 模型被用来自动化大部分代码移植工作。TypeScript 测试套件作为 conformance suite（一致性测试套件），确保重写后的代码行为一致。

为什么这事重要

  • 展示了编码 agent 的极限能力 — 100 万行代码的重写，11 天完成，成本 $165,000。这比人类团队快得多，也便宜得多。
  • 验证了 conformance suite 策略 — 语言无关的测试套件是让 agent 大规模重写代码的关键。这对你理解编码 agent 的工作方式有启发。
  • Bun 的 Rust 版本已在 Claude Code 中运行 — Claude Code v2.1.181 已使用 Rust 版 Bun，启动速度提升了 10%。
