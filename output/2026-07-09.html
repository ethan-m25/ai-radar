Grok 4.5 发布：编码 Agent 降价战开打

clawii，xAI 今天发布了 Grok 4.5，一个 1.5 万亿参数的编码 Agent 模型，定价 $2/$6 每百万 token，比 Opus 4.8 便宜至少 5 倍。Cursor 参与了训练，已经宣布支持。这对你选编码 Agent 模型有直接影响——又多了一个选择，而且价格打下来了。

📍 背景 · 编码 Agent 模型市场正在快速分化：OpenAI 有 GPT-5.5/5.6 Codex，Anthropic 有 Opus 4.8/Sonnet 5，开源有 GLM-5.2/Ornith。xAI 的 Grok 之前主打对话，4.5 版本首次聚焦编码和 Agent，强调性价比而非绝对分数。Elon Musk 称其为“Opus 级别”但更快、更便宜。Cursor 在训练过程中深度合作，证明其编码能力已经过实战检验。

为什么这事重要

  • 价格冲击 — $2/$6 的价格远低于 Opus 4.8 ($15/$75) 和 GPT-5.5 ($10/$50)，接近 Sonnet 5 的促销价 ($2/$10)。这意味着你可以在不牺牲太多能力的情况下大幅降低 Agent 使用成本。
  • Cursor 背书 — Cursor 称其为“最强模型”，并已集成。如果你用 Cursor，可以直接在 settings 里切换 Grok 4.5 试试。
  • 1M 上下文 — 上下文窗口很快恢复到 1M token，适合处理大型代码库。
  • 生态支持 — 已接入 Grok Build/API、Hermes Agent、Portal、OpenRouter，可以无缝替换现有模型。
🎯 今天就能做的事
  1. 如果你用 Cursor，打开 Settings → Models → 选择 Grok 4.5，立刻体验编码能力。
  2. 比较价格：在 OpenRouter 上对比 Grok 4.5 和 Opus 4.8 / Sonnet 5 的成本。
  3. 用你的典型编码任务测试一下，看它是否真的“Opus 级别”。

OpenAI GPT-Live 语音模式升级

clawii，OpenAI 今天升级了 ChatGPT 语音模式，新模型 GPT-Live 质量大幅提升，还能在后台调用 GPT-5.5 处理复杂任务。Simon Willison 试用了一小时，说这是“最有用的语音模式”。虽然你主要用编码，但日常查询、头脑风暴时语音可能比打字更快。

📍 背景 · ChatGPT 语音模式之前基于 GPT-4o 时代模型，知识截止 2024 年，能力有限。GPT-Live 使用了更新的模型，支持实时对话、打断、情绪识别，还能将复杂任务（搜索、推理）委托给 GPT-5.5 在后台处理，同时保持对话流畅。Simon Willison 在评测中表示“之前很少用语音，现在可能会经常用”。

为什么这事重要

  • 模型质量大幅提升 — 不再是“愚蠢的语音助手”，能理解复杂问题，甚至能跟你讨论编程思路。
  • 后台委托机制 — 语音模式可以触发 GPT-5.5 做搜索、推理，结果异步返回，不影响对话流。这有点像多 Agent 的雏形。
  • 对你日常有用 — 走路、开车、做家务时可以用语音讨论编程问题，模型会记住上下文，继续之前的话题。
🎯 今天就能做的事
  1. 打开 ChatGPT 移动 App，启用语音模式，体验 GPT-Live。
  2. 尝试用语音讨论一个编程问题，看它是否能理解并给出有价值建议。
  3. 注意观察它是否会自动切换到后台模型处理复杂任务。

老实说：语音模式仍然不适合所有场景，比如安静办公室或需要精确代码的场景。但在移动中，它比打字快很多。

Bun 重写案例：Agent 工程极限展示

clawii，Simon Willison 今天详细拆解了 Bun 从 Zig 到 Rust 的完整重写过程，使用了 Claude Code/Fable，成本 $165k，生成 5.9B 输入 token、690M 输出 token。这不是一个可以直接用的工具，但展示了当前 Agent 能做到的极限：10 天内重写一个大型运行时，测试通过率极高。

📍 背景 · Bun 是一个 JavaScript 运行时（类似 Node.js），之前用 Zig 写。Jarred Sumner（Bun 作者）决定用 Claude Code 重写为 Rust，因为 Zig 的内存管理问题导致大量 bug。整个重写过程利用了 TypeScript 测试套件作为 conformance suite，Agent 自动翻译代码，并通过对抗性审查确保质量。最终 PR 有 +1M 行代码，但上线后几乎无人察觉——除了启动速度快了 10%。

为什么这事重要

  • Agent 能力的新维度 — 这不是简单补全代码，而是完整的语言重写，涉及大量逻辑转换和边界情况。
  • 测试套件是关键 — 语言无关的测试套件让 Agent 可以自动验证重写是否正确，这是大规模 Agent 项目的基石。
  • 成本门槛 — $165k 的 API 成本（内部价更低）说明这种规模的 Agent 项目仍需要巨大投入，但门槛在快速下降。
