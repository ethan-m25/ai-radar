Better Models: Worse Tools — 新 Claude 模型 tool call 格式退化

clawii，如果你用 Pi 或者其他非 Claude Code 的 agent 框架调用 Claude，可能会发现 Opus 4.8 和 Sonnet 5 的 tool call 反而比旧模型更差。Armin Ronacher 在博客里详细记录了这个现象。HN 上拿了 180 点。

📍 背景 · Armin 在开发 Pi 工具时发现，Opus 4.8 和 Sonnet 5 在调用自定义 edit 工具时会凭空添加不存在的字段，导致工具调用被拒绝。旧模型（如 Sonnet 4.5、Opus 4.5）反而更干净。他推测这是因为 Anthropic 对新模型进行了 RL 训练，让它们更擅长 Claude Code 内置的 search‑and‑replace 工具，而其他框架的定制工具则被“污染”了。这个问题在 HN 上引发了大量讨论，很多独立 agent 框架的开发者都遇到了。

为什么这事重要

  • 直接影响你用的 agent 工具 — 如果你用 Pi、Continue、或其他基于 Claude 的 agent，新模型可能不如旧模型好用。这会影响你的 vibe‑coding 效率。
  • 反映了一个结构性问题 — 模型公司为自家工具做 RL 优化，导致第三方框架的兼容性下降。这不是 bug，是激励不一致。
  • 你可以立刻应对 — 暂时降级到 Sonnet 4.5 或 Opus 4.5，或者改用 Claude Code 自带的编辑工具。
🎯 今天就能做的事
  1. 如果你用 Pi 或其他非 Claude Code 框架，将模型回退到 Sonnet 4.5 或 Opus 4.5，观察 tool call 是否变干净。
  2. 阅读 Armin 的博客文章 Better Models: Worse Tools，了解具体细节。
  3. 如果你维护自己的 agent harness，考虑为不同模型准备多套 edit 工具格式，或者实现自动检测。

GPT-5.5 Codex 推理 token 聚类导致性能下降

clawii，HN 上一篇文章指出 GPT-5.5 Codex 的推理 token 聚类机制可能导致编码 agent 的输出质量下降。287 点，114 条评论。如果你在用 Codex，这事直接关系到你的工作流。

📍 背景 · OpenAI 在 GPT-5.5 Codex 中引入了一种称为“推理 token 聚类”的优化，目的是减少 token 用量和提高速度。但社区发现，这种聚类有时会破坏代码上下文的连贯性，导致生成的代码质量不如旧版本。具体表现为：变量名错误、逻辑跳跃、忽略上下文中的关键细节。这不是一个简单 bug，而是优化权衡的副作用。

为什么这事重要

  • 直接影响你日常使用的 Codex — 你每天都在用 Codex 写脚本、处理数据。如果输出质量下降，你花在调试和修正上的时间会显著增加。
  • 可能是一个隐藏的成本增加 — 质量下降意味着你需要更多次迭代（更多 token 消耗），最终可能让 Codex 的所谓“效率提升”得不偿失。
  • 社区已经开始寻找 workaround — 包括降级到 GPT-5.5 之前的版本、调整 prompt 结构等。你需要知道这些策略。
🎯 今天就能做的事
  1. 检查你的 Codex 版本，如果使用的是最新 GPT-5.5，可以尝试显式指定使用旧版本模型（如果 API 支持回退）。
  2. 阅读 HN 上的讨论 GPT-5.5 Codex reasoning-token clustering，了解具体的退化案例。
  3. 在你的日常工作流程中，增加一个质量检查步骤：在 Codex 输出后，用另一个模型（如 Fable 5）做快速复核。
🔬 深扒——推理 token 聚类是什么？为什么会导致退化？
方面正常推理推理 token 聚类优化
机制模型为每个 token 独立计算注意力将相邻的推理 token 合并成簇，一次性处理以减少计算量
优点无信息损失速度提升约 20-30%，token 用量减少
缺点无上下文细节可能被聚类丢失，尤其是跨多行的代码逻辑
受影响场景所有编码任务长上下文、需要精确变量引用的任务最严重

对你最相关的：如果你经常让 Codex 修改已有文件（比如添加功能、重构代码），聚类优化可能会导致它“忘记”之前定义过的变量或函数。

老实说：这个退化是渐进式的，不是每个人都会遇到。如果你主要在写短脚本或简单函数，可能感觉不到。但如果你是长上下文 vibe‑coding，建议降级或使用其他模型。

🔭 关注点 · OpenAI 是否会在后续更新中修复（可能通过更智能的聚类策略）？社区是否会开发检测工具？如果退化被广泛证实，可能会促使更多人转向开源模型。
术语解码 · 推理 token 聚类（reasoning‑token clustering） = 将多个连续的推理思考 token 合并成组，一次性处理以节省计算资源。注意力（attention） = 模型关注输入序列中不同部分的能力。
