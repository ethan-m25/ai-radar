GLM 5.2 发布——智谱开源模型，HN 608 分，社区热评

智谱在 Twitter 上宣布了 GLM 5.2，HN 冲到 #4（608 分），336 条评论。这不是一个“又一个中国模型”——社区反馈非常正面，很多人说它在推理和代码任务上“出乎意料地好”。对于你这种用 vibe coding 做办公自动化的场景，GLM 5.2 可能是一个值得关注的替代品，尤其是如果你对 Fable 5 的出口管制感到不安。

📍 背景

GLM 系列是智谱（Zhipu AI）的开源大语言模型系列。GLM 5.2 是继 GLM 5.1 之后的最新版本。智谱是中国头部 AI 实验室之一，与清华大学有深度合作。GLM 系列一直以“中文友好 + 开源”著称。这次 GLM 5.2 的发布在 HN 上获得如此高的分数（608 分，#4），说明它已经引起了国际社区的关注——而不仅仅是中文圈。

为什么这事重要

  • HN 608 分不是闹着玩的 — 这是一个中国模型在 HN 上获得的罕见高分。336 条评论中，很多来自实际使用过的开发者，评价正面。
  • 对 Fable 5 出口管制的直接替代 — 如果你因为 Fable 5 被切断而在寻找替代品，GLM 5.2 是一个值得认真考虑的选项。它是开源的，不受美国出口管制。
  • 中文能力天然强 — 对于你的办公场景（中文文档、Excel 自动化、邮件撰写），GLM 的中文能力可能比同尺寸的英文模型更好。
  • 开源意味着你可以自己部署 — 不需要依赖 API 提供商，没有被切断的风险。

  4. 关注智谱的官方公告 — 查看智谱的 Twitter 或官方博客，了解 GLM 5.2 的具体参数和开源协议。

OpenAI Codex 开源计划——代码 agent 的底层基础设施要开放了

OpenAI 在 HN 上宣布了“Codex for open source”计划（237 分，#17）。这不是 Codex 模型本身开源，而是 Codex 的底层基础设施和工具链——包括 agent 框架、sandbox 环境、MCP 服务器等——将开源。对于你这种用 Claude Code 和 Codex 做 vibe coding 的人来说，这意味着未来你可以更自由地定制和扩展你的 coding agent 环境。

📍 背景

Codex 是 OpenAI 的 AI coding agent 产品，与 Claude Code 和 Cursor 竞争。Codex 的核心是 GPT-5.5-Codex 模型，但 OpenAI 一直在构建围绕它的工具链：agent 框架、sandbox 环境、MCP 服务器集成等。这次“Codex for open source”计划意味着 OpenAI 将把这些基础设施组件开源，让社区可以自由使用和修改。这与 Anthropic 的 Claude Code 策略不同——Anthropic 一直保持 Claude Code 的闭源。

为什么这事重要

  • OpenAI 在走“开源基础设施”路线 — 与 Anthropic 的闭源策略形成对比。如果你用 Claude Code，你可能已经感受到了“被锁定”的感觉。Codex 的开源计划给了你另一个选择。
  • 对 vibe coding 生态的影响 — 如果 Codex 的 agent 框架和 sandbox 环境开源，社区可以基于它们构建自己的工具。这可能会加速整个 coding agent 生态的发展。
  • 对 Fable 5 出口管制的间接影响 — 如果你因为 Fable 5 被切断而考虑迁移到其他平台，Codex 的开源计划可能是一个加分项。虽然 Codex 模型本身不是开源的，但至少底层工具链是开放的。
  • HN 237 分，#17 — 虽然不是最高分，但说明社区对这个消息有相当的兴趣。
🎯 今天就能做的事
  1. 阅读 HN 讨论 — HN 链接。了解社区对 Codex 开源计划的反应。
  2. 关注 OpenAI 的官方公告 — 查看 OpenAI 官方页面，了解开源的具体范围和时间表。
  3. 对比 Claude Code 和 Codex — 如果你正在选择 coding agent 平台，考虑以下因素：Claude Code 闭源但模型强（Fable 5 已切断）；Codex 模型闭源但工具链开源。你的选择取决于你更看重模型能力还是生态开放性。
  4. 关注后续的 GitHub 仓库 — OpenAI 可能会在 GitHub 上发布开源组件。关注 OpenAI GitHub 组织。

具体开源什么？ — 根据 HN 帖子和 OpenAI 官方公告，Codex 开源计划包括：1）agent 框架（用于构建和编排 coding agent）；2）sandbox 环境（安全的代码执行环境）；3）MCP 服务器集成（与外部工具交互）；4）开发者工具和 CLI。注意：Codex 模型本身（GPT-5.5-Codex）不开源。

对你意味着什么 — 如果你用 Claude Code 做 vibe coding，你可能已经习惯了 Anthropic 的闭源生态。Codex 的开源计划意味着你可以：1）在 Codex 的框架上构建自定义工具；2）使用 Codex 的 sandbox 环境运行 Claude Code（如果兼容）；3）减少对单一供应商的依赖。

诚实 caveat — 1）开源的具体时间表尚未公布；2）开源的范围可能有限（不是全部 Codex 组件）；3）Codex 模型本身仍然闭源，这意味着最核心的部分你无法控制。
