Claude Opus 4.8 + Dynamic Workflows — 模型升级，但真正的新闻是并行 agent 编排

Anthropic 今天发布了 Opus 4.8 和 Dynamic Workflows。Opus 4.8 在 SWE-Bench Pro 上拿到 69.2%（比 GPT-5.5 高 10 分），价格不变，Fast 模式 2.5x 更快。但 Dynamic Workflows 才是真正改变游戏规则的东西——Claude 现在可以自己写编排脚本，同时启动数百个并行子 agent 协作完成大型任务。这直接改变了你用 Claude Code 能做的事。

📍 背景 · Context

Opus 4.7 是 Anthropic 在 2026 年初发布的旗舰模型，在 coding 和 agent 任务上表现强劲但被一些用户抱怨"懒"——提前停止任务、过度自信地报告进度、不自觉地截断复杂工作。Opus 4.8 直接回应这些反馈：更诚实（主动说不知道）、更持久（不会提前停）、更细致。与此同时，Dynamic Workflows 是 Claude Code 中启用的新编排系统——本质上是一个"递归 agent"框架，Claude 自己写 orchestration 脚本然后 spawn 大量子 agent 并行工作。这被一些研究者视为 Recursive Language Models 概念的产品化。

为什么这事重要

  • Dynamic Workflows 让你能用数百个并行 agent 协作——Claude 自己写编排脚本，严格遵循，然后启动成百上千个子 agent 并行处理任务。Anthropic 展示的例子：把 Bun 从 Zig 移植到 Rust，75 万行代码，11 天从第一次 commit 到合并，99.8% 测试通过。另一个例子：10 分钟内并行处理数百个 A/B 测试 flag，识别出过期的。对你来说，这意味着以前需要几天拆解的大任务，现在可以丢给 Claude 让它自己拆。
  • Opus 4.8 模型本身更强了——SWE-Bench Pro 69.2%（比 GPT-5.5 高 10 分），FrontierSWE #1，APEX-SWE 45.3% Pass@1（比 GPT-5.3 Codex 高近 4 分）。显著提升：Terminal-Bench Hard +6.8，τ²-Bench Telecom +5.9，IFBench +3.6。更重要的是，官方说它用 15% 更少的轮次和 35% 更少的输出 token 达到了更高的性能。
  • 价格不变，Fast 模式 2.5x 更快——输入 $5/M token，输出 $25/M token。1M 上下文窗口。Fast 模式比 Opus 4.7 Fast 快 2.5 倍，而且只比普通 4.8 贵 2 倍（之前 Fast 模式比普通贵 6 倍）。经济性大幅改善。
  • 模型更诚实了——社区一致反馈：Opus 4.8 更愿意说"我不知道"，主动标记代码中的问题，避免虚假的任务完成报告。这解决了很多 heavy user 对 Claude 的最大抱怨。

  1. 打开 Claude Code 试试 Dynamic Workflows（5 分钟）——在 Claude Code 里输入包含 "workflow" 关键词的 prompt 来激活。比如 "write a workflow to refactor this codebase" 或 "create a workflow to analyze all these logs"。观察 Claude 是否自动生成编排脚本并启动子 agent。
  2. 切换到 Opus 4.8（3 分钟）——在 Claude Code 配置或 Claude.ai 的模型选择中切换到 Opus 4.8。价格和 4.7 一样，但性能和效率都更好。
  3. 尝试 Fast 模式——如果你之前因为价格没怎么用 Fast 模式，现在它相对普通模式只贵 2 倍（而不是 6 倍），值得在日常简单任务上试试。
  4. 读 Dan Shipper 的评测——Dan Shipper 说 Opus 4.8 "could've been called Opus 5"。他推荐 coding 用 xhigh effort、writing 用 high effort。这可以帮你快速找到最有效的使用方式。

🔬 深扒 — Dynamic Workflows 到底怎么工作 + 代价是什么

Dynamic Workflows 的工作方式：

  • 你在 Claude Code 里给出一个大型任务（例如"把这个代码库从 JS 移植到 Python"）
  • Claude 自己写一个 orchestration 脚本，描述任务拆解和子 agent 分配方案
  • Claude 严格遵循这个脚本，同时启动数百个并行子 agent
  • 每个子 agent 处理一个子任务，两个 reviewer agent 审查每个文件的输出
  • 结果汇总后返回给你

vs 你之前手动拆任务：以前你需要自己决定怎么拆任务，一个个按顺序让 Claude 做。现在 Claude 自己拆、自己排、自己并行执行。这是从"你用工具"到"你告诉工具目标，工具自己规划执行"的转变。

代价：

  • token 消耗非常大——数百个并行 agent 意味着大量推理 token。社区反馈："quota-burning"、"token-mogged by GPT-5.5"。
  • 并行编辑可能有冲突——Theo 抱怨过并行编辑的冲突问题。
  • 需要 Max/Team/Enterprise 计划或 API 访问——不是所有层级都可用。

诚实 caveat：Dynamic Workflows 是 research preview，不是正式版。token 成本没有官方估算，实际使用中可能比手动拆分更贵。建议先在中小型任务上测试，不要一上来就丢 75 万行代码给它。

🔭 接下来看什么

三个关键触发点：(1) Dynamic Workflows 从 research preview 转为正式版——那时可以放心在生产中使用；(2) Anthropic 提到计划发布"比 Opus 更智能的新模型类别"（推测是 Mythos 类模型），在更强的安全保障后推出——这将是 🚨 级别信号；(3) 社区发布 Dynamic Workflows 的 token 成本分析——决定是否值得在常规工作中使用。

👀 Worth a Look · 产品更新

Mistral Vibe 更新 — 新增 Work/Code 模式 + VS Code 扩展

Mistral 今天给 Vibe 加了 Work 和 Code 两种模式，还推出了 VS Code 扩展。Vibe 是 Mistral 的统一 agent 产品，对标 Claude Code 但更偏通用生产力。如果你用 VS Code，又多了一个直接在编辑器里使用的 AI agent 选择——而且 Mistral 的定价通常比 Anthropic/OpenAI 便宜。

📍 背景 · Context

Mistral 是法国 AI 公司，以开源模型（Mistral Small/Medium/Large）和高效率著称。Vibe 是他们的 agent 产品线——2026 年初推出的"远程 agent"产品，可以运行在云端、执行长期任务。之前 Vibe 主要面向编码场景。今天的更新把 Vibe 拆成两个模式：Work 模式（文档分析、数据整理、知识工作等非编码任务）和 Code 模式（编码 agent）。同时发布 VS Code 扩展，让你在编辑器里直接使用 Vibe。基于 Mistral Medium 3.5 模型（5 月底发布的新模型）。

为什么这事重要

  • Vibe 现在覆盖了非编码场景——Work 模式让你可以用 Vibe 做文档分析、报告写作、数据整理等办公室工作。对 clawii 来说，这可能是 Claude.ai 或 ChatGPT 之外的另一个日常 AI 帮手，而且背后的 Mistral 模型以效率著称（同样任务用更少 token）。
  • VS Code 扩展 = 直接在编辑器里用——和 Claude Code（终端内）不同，Vibe 的 VS Code 扩展是完整的 GUI 体验。如果你习惯 VS Code 界面而不是终端，这可能更舒服。
  • Mistral 的定价通常更低——Mistral Medium 3.5 的 API 价格比 Opus 便宜很多。如果你对成本敏感，Vibe 可能是一个经济实惠的替代方案。
  • 更多竞争 = 更好选择——Anthropic（Claude Code）、OpenAI（Codex/ChatGPT）、Mistral（Vibe）三家都在卷 agent 产品。竞争会推动价格下降和功能提升，最终受益的是用户。
🎯 实操 — 今天能做什么
  1. 读 Mistral 的公告（5 分钟）——mistral.ai/news 上的 "Vibe gets to work" 文章。了解 Work 和 Code 模式的具体区别和定价。
  2. 安装 VS Code 扩展（2 分钟）——在 VS Code 扩展市场搜索 "Mistral Vibe" 安装。试试在编辑器里直接提一个简单的编码或文档处理任务。
  3. 和 Claude Code 对比——同一个任务分别用 Claude Code 和 Vibe 做一次，比较速度、质量和成本。Mistral 模型通常在效率上有优势（更少的 token 消耗）。
  4. 别着急切换主力工具——先试用，等 1-2 周看看社区反馈再决定是否作为日常工具。Mistral 的生态（插件、社区支持、集成）还比不上 Anthropic 和 OpenAI。

对 clawii 的实际意义：

  • Vibe 的核心优势是 Work 模式——Claude Code 和 Codex 都是代码优先。Vibe 的 Work 模式覆盖了文档处理、数据分析等办公室工作，这让它成为一个更通用的 AI 工具。
  • Mistral 的效率优势——如果你对 token 成本敏感，Mistral 模型通常用更少的 token 完成同样任务。长期使用可能省不少钱。
  • VS Code 扩展降低了使用门槛——不需要用终端，直接在 VS Code 里操作，对非开发者更友好。

诚实 caveat：Mistral 的生态远不如 Anthropic 和 OpenAI。社区插件少、第三方集成有限、企业级支持不如大厂。作为主力工具还需要观望。另外底层模型 Mistral Medium 3.5 在纯编码能力上是否比得上 Opus 4.8 或 GPT-5.5 Codex 还有待验证。
