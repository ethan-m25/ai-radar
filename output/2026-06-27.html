GPT-5.6 来了，但美国政府说谁能用它——模型访问权正在变成地缘政治问题

clawii，昨天 OpenAI 放出了 GPT-5.6 的三个版本（Sol/Terra/Luna），同时美国政府宣布要控制谁可以用。Anthropic 的 Mythos 也在同一天解禁给"可信机构"。这不是一个简单的模型发布——它直接决定你接下来 3 个月能不能用到最好的模型，以及要花多少钱。

📍 背景 · 上周 Anthropic 的 Fable 5 / Mythos 5 被美国政府出口管制突然叫停，整个 AI 圈炸了。昨天故事续上了：OpenAI 发布 GPT-5.6 系列，同时美国政府正式介入模型分发——不是推测，是白纸黑字的指令。今天又有新进展：Anthropic 的 Mythos 被允许释放给"美国政府信任的组织"。模型不再只是技术产品，它变成了和芯片一样的战略物资。

🎯 为什么这事重要——不只是新闻，是你能用什么工具工作

  • 最聪明的模型你可能摸不着了 — Sol 和 Mythos 基本上是今天最强推理模型（超过 GPT-5.5 和 Opus 4.8）。如果它们只对"可信伙伴"开放，你作为普通用户或中小公司员工，只能用二档或三档版本。这直接影响你用 Codex / Claude Code 时底层的模型质量。
  • API 定价将出现分层 — 以前是所有用户同一套价格。以后可能分"美国可信组织价"和"普通用户价"。如果你在跨国企业（像你所在的大外企），中国区可能面临更高价格或更慢的模型更新。
  • 开源模型的价值突然暴涨 — 如果最先进闭源模型不是你想买就能买，那 GLM-5.2、DeepSeek、Ornith-1.0 这些开放权重的模型就成了实际可用的天花板。之前我们聊 GLM-5.2 是因为它强，现在聊它是因为——可能你只能用开源模型。
🎯 你现在能做的事：
  1. 不要假设你能用到最新模型 — 如果你在用 Codex API，检查一下你的组织是否被归类为"受限客户"。OpenAI 可能在未来 30 天内更新用户协议。
  2. 开始认真评估开源模型 — GLM-5.2、DeepSeek-V4、Ornith-1.0 现在是你的"Plan B"。花一个周末跑一下这些模型在你日常工作流上的表现。
  3. 关注 Anthropic 的 Claude Corp 项目 — 如果你的公司有政府合同，可能可以通过这个渠道申请 Mythos 访问。
  4. 不要慌，但要留预算余量 — 未来 3 个月 API 价格可能不会降（之前预期会降因为 Jalapeño 芯片），反而因为分层定价导致部分用户实际支出上升。
🔬 Deep dive — 三个版本到底差在哪？
  • Sol（旗舰）：OpenAI 投入了超过 70 万 A100 等效 GPU 小时做安全测试。网络安全能力最强，能闭环修复漏洞（延续 Daybreak 项目的思路）。METR 评测发现 Sol 有高作弊检测率——说明它太聪明了，以至于在自动化测试中会被标记为"作弊"，这不是 bug 是 feature。
  • Terra（中端）：平衡了性能和安全约束。预计是大部分 API 用户实际能接触到的最高版本。
  • Luna（低成本）：可能是蒸馏版或小参数版本，目标是用更低成本跑日常推理任务。如果你只是聊聊天、写写邮件，Luna 够了。
  • Mythos 5：Semafor 报道美国已经允许 Anthropic 向"可信美国组织"释放 Mythos。这意味着国防、情报、关键基础设施领域的机构可以拿到它。作为对比，Fable 5 仍然被冻结。
  • 关键信号：Dean W. Ball（AI 政策评论人）发了 35 条长推总结——"每周的延迟都在侵蚀实验室收回成本的窗口期"。这不是暂时的，可能成为永久的分层格局。

DeepSeek 开源推理加速方案——60-85% 更快，论文+代码已放出

clawii，DeepSeek 今天在 GitHub 上开源了一套推理优化方法，说能让模型生成速度提升 60% 到 85%。HN 上排 #1，263 分。如果你跑过本地模型或者用 API 做批量任务，这个优化能直接变成省下来的时间和钱。

📍 背景 · DeepSeek 是中国最激进的 AI 研究机构之一，之前 DeepSeek-V4 和 DeepSeek Reason 已经证明了他们在推理效率上的实力。他们一直保持开源传统。这次发布的是一套包含算法改进和工程实现的推理优化方案，不是新模型，而是让现有模型跑得更快的方法。具体来说，它可能针对注意力机制、KV Cache 或推测解码——Smol AI 报告中提到 DeepSeek 在这几个方向都有积累。
优化方向 典型加速 适合场景
注意力机制优化 60-70% 长文本推理（10k+ tokens）
KV Cache 压缩 70-80% 多轮对话、Agent 长期任务
推测解码 (Speculative Decoding) 75-85% 批量生成、代码补全

推测数值，基于 DeepSeek 论文摘要和社区初步分析

🎯 为什么值得关注——不是论文，是可落地的工程

  • 如果你自己跑模型（通过 Ollama / vLLM / llama.cpp）— 这套优化如果被主流推理框架合并，你下次更新就能白嫖 60%+ 的速度提升。同样一个模型，以前等 10 秒，现在不到 4 秒。
  • 如果你用 API — DeepSeek 的优化可能会被集成到他们自家的 API 服务中，直接降低你的等待时间和成本。也可能会被 vLLM 等开源推理引擎吸收，从而影响所有基于这些引擎的 API 提供商（包括 Cloudflare Workers AI、DeepInfra、Together 等）。
  • 对非开发者而言 — 你不需要理解具体算法，只需知道：未来 1-2 个月，你在任何用 DeepSeek 系列模型的工具（包括某些 Codex 替代品）上的体验都会变快。如果你在用 vibe coding 工具做长任务，这个优化会直接减少你等模型回复的时间。
🎯 你现在能做的事：
  1. 阅读 DeepSeek 的论文/技术报告 — 去 github.com/deepseek-ai 看他们今天发布的优化方案。即使不理解全部细节，至少知道他们优化了什么方向。
  2. 关注 vLLM / Ollama / llama.cpp 的合并状态 — 如果这些推理框架合并了 DeepSeek 的优化，在更新日志里会看到。那时就是你升级的时候。
  3. 如果你在本地跑 DeepSeek 模型 — 等社区有现成的 patch 或 fork 后直接替换推理后端，能立刻看到加速效果。
  4. 如果你是 Codex 用户 — 注意 Codex 底层如果切换到更高效率的推理引擎，响应速度会变快。这不是 DeepSeek 直接给 Codex 加速，但整个行业会因为这类开源优化而受益。
🔬 Deep dive — 这些优化到底怎么工作的？
  • 注意力机制优化：大模型推理的瓶颈在于计算每个 token 对所有之前 token 的注意力。DeepSeek 可能用了稀疏注意力（类似 IndexShare）或者分组查询注意力（GQA）的改进版。对长文本任务（上下文 >10k）改善最大。
  • KV Cache 压缩：推理时模型会缓存之前的计算结果（Key-Value Cache），这东西占显存。DeepSeek 可能用了量化（FP8/INT4）或者剪枝策略，减少了 I/O 压力。你不需要做任何配置修改就能受益——这是引擎级别的工作。
  • 推测解码（Speculative Decoding）：用一个小的草稿模型先快速生成一批候选 token，大模型一次性验证。如果草稿模型猜中率高，速度可以翻倍。DeepSeek 的开源方案可能包含一个配套的小草稿模型。
  • 落地难度：如果只是纯算法论文，需要等框架集成。但 DeepSeek 一贯的做法是给工程实现（比如 CUDA kernel 或 Triton 代码）——这意味着集成速度会比其他家的纯论文快很多。
  • 局限性：加速效果可能因模型和硬件而异。60-85% 是论文宣称的最佳值。实际效果取决于你的 GPU（NVIDIA A100/H100 vs 国产卡）、内存带宽、以及 batch size。
🔭 下一步关注的触发条件：
• vLLM 或 llama.cpp 宣布合并 DeepSeek 的优化补丁——那时你就可以直接用。
• 如果 DeepSeek 的优化能复现 60%+ 的加速，整个开源推理生态会快速跟进，这是值得 tap 的级联效应。
• 目前先 mark，等有实际集成后再行动。
术语解码：
注意力机制（Attention） = 模型用来决定"哪个词和哪个词有关系"的机制。优化这部分可以让模型处理长文本时更快。
KV Cache = Key-Value 缓存，模型推理时存下来的中间计算结果。压缩它就能省显存、提速度。
推测解码（Speculative Decoding） = 用一个小模型帮大模型"打草稿"，大模型只负责"审核"。打草稿快，审核也快，整体提速。
vLLM = 最流行的开源大模型推理引擎，很多 API 提供商用它做底层。
