GPT-5.6 正式发布 — 三档定价，全面可用

clawii，OpenAI 今天正式发布了 GPT-5.6 系列，包括 Sol（旗舰）、Terra（中档）和 Luna（低成本）三个变体。Sam Altman 称 Sol 是"我们生产过的最好的模型"。定价从 $1 到 $5 每百万 token，还有 90% 的缓存读取折扣。Simon Willison 的博客给出了详细的评测和对比。

📍 背景 · GPT-5.6 预览版在 6 月 26 日首次亮相，当时受美国政府限制仅对可信合作伙伴开放。今天正式发布意味着全面可用，包括 ChatGPT、Codex 和 API。新功能包括 ChatGPT Work（桌面应用合并 Codex 和 ChatGPT）、Sites beta、程序化工具调用和多 agent beta。定价方面，Sol 输入 $5/M、输出 $25/M；Terra 输入 $2/M、输出 $10/M；Luna 输入 $1/M、输出 $5/M。缓存读取 90% 折扣。

为什么这事重要

  • 全面可用意味着你可以直接用了 — 之前预览版需要申请，现在通过 API 和 ChatGPT 都能访问。如果你用 Codex，Sol 是默认模型。
  • 定价有竞争力 — Sol 比 Opus 4.8 便宜（$5/$25 vs $15/$75），Terra 和 Sonnet 5 同价（$2/$10），Luna 是最便宜的旗舰级模型。
  • Simon Willison 的评测显示 Sol 在编码 agent 上接近 Fable 5 — Intelligence Index 得分 59，略低于 Fable 5 但成本只有三分之一。

🎯 今天就能做的事
  1. 在 OpenAI API 上切换模型到 gpt-5.6-sol 或 gpt-5.6-terra，对比编码效果。
  2. 如果你用 Codex，它应该已经默认使用 Sol。试试用 Sol 跑一个你之前用 Sonnet 5 做过的任务，看质量差异。
  3. 算一下成本：如果你每天用 10M 输入 + 2M 输出，Sol 每天约 $100，Terra 约 $40，Luna 约 $20。相比 Opus 4.8 的 $300，省很多。

对你最相关的：如果你用 Codex，Sol 是默认模型，性能接近 Fable 5 但成本低很多。如果你用 Claude Code，Sonnet 5 仍然是默认，但你可以通过 API 调用 GPT-5.6 做对比。

老实说：Simon Willison 的评测显示 Sol 在编码 agent 上得分 59，Fable 5 更高但成本是 3 倍。Terra 和 Luna 是性价比选择。但注意 Sol 的上下文只有 128K，比 Sonnet 5 的 1M 小很多，不适合超大项目。

🔭 关注点 · GPT-5.6 Sol 在 SWE-bench 等独立编码 benchmark 上的表现。社区对 Sol 和 Fable 5 的实际对比。缓存折扣对实际成本的影响。如果 Sol 在编码 agent 上接近 Fable 5，OpenAI 的定价策略会改变编码 agent 的经济模型。
术语解码 · Token = 模型处理的最小单位，大约 1 个英文单词 ≈ 1-2 个 token，1 个中文字 ≈ 2-3 个 token。缓存读取 = 当重复使用相同输入时，模型可以读取缓存结果，大幅降低成本。GPT-5.6 的 90% 折扣意味着缓存命中时价格只有原来的 10%。

Muse Spark 1.1 — Meta 开放 API，工具调用改进

clawii，Meta 发布了 Muse Spark 1.1，这是第一个提供 API 的 Spark 模型。Simon Willison 说它在 agentic tool calling 和 computer use 上有显著改进。他做了一个 LLM 插件让你在命令行里用这个模型。

📍 背景 · Muse Spark 是 Meta 在 4 月发布的轻量级模型，主打高效推理。1.1 版本首次开放 API，并改进了工具调用和计算机使用能力。Meta 还发布了详细的评估报告，包括有趣的"自对话吸引子状态"现象。

为什么这事重要

  • 又一个可用的编码 agent 模型 — 虽然 Muse Spark 不是专门为编码设计的，但工具调用改进意味着它可以用于 agent 任务。如果你在 Meta 生态里，值得一试。
  • Simon Willison 做了 LLM 插件 — 你可以用 uv tool install llm 和 llm install llm-meta-ai 快速在命令行里使用。
  • 定价未知 — API 价格尚未公布，但预计会走性价比路线。
