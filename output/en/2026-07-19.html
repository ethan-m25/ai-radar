Claude Code now uses Bun (Rust-based runtime)

Hey clawii — Simon Willison reported today that Claude Code has switched its JavaScript runtime from Node.js to Bun, which is built with Rust (and Zig). This isn't a language change for Claude Code itself — it's the underlying runtime. Bun is significantly faster than Node.js, especially on startup and package management. For your daily vibe-coding sessions, this means snappier responses and a smoother experience.

📍 Context · Claude Code is Anthropic's terminal-based coding agent. It was originally built on Node.js. Bun is an emerging JavaScript runtime written primarily in Zig (with some Rust components) that's known for ~5-10x faster startup times and a built-in package manager. Anthropic switching runtimes is a non-trivial engineering move — it signals they're serious about optimizing the developer experience.

Why this matters

  • Daily experience improvement — Bun starts 5-10x faster than Node.js. If you open Claude Code dozens of times a day, those seconds add up.
  • Anthropic cares about performance — Switching runtimes isn't easy. This shows they're actively optimizing Claude Code's UX.
  • Bun ecosystem validation — Adoption by Anthropic is a major endorsement. If you work with JavaScript/TypeScript, Bun is worth trying.
🎯 What you can do today
  1. Update Claude Code: npm update -g @anthropic-ai/claude-code (if installed via npm) or re-download the CLI.
  2. Test startup speed: run claude and notice if it feels faster.
  3. Try Bun yourself: curl -fsSL https://bun.sh/install | bash, then bun run your-script.js.

Transcribe.cpp + Moonshine — local speech recognition hits new low bar

Hey clawii — Two local speech recognition projects blew up on HN today: Transcribe.cpp (551 points) and Moonshine (468 points). Transcribe.cpp is a lightweight C++ speech-to-text tool. For you, this means you can do speech-to-text locally without relying on OpenAI Whisper API or Google Cloud.

  • Potential vibe-coding use case — Voice input for coding or voice-controlled agents. Not mainstream yet, but the tools are now mature enough to experiment with.
  • Privacy win — Everything runs locally. No audio uploaded to the cloud.
