👀 Worth a Look · Enterprise Cost Trend

Uber caps AI tools at $1,500/month per employee — the new enterprise budget norm

Uber blew through its whole-year AI budget in four months and is now limiting each employee to $1,500/month per AI coding tool (Claude Code, Cursor, etc.). Simon Willison did the math: for a software engineer, that's ~11% of median compensation. For clawii, this is the most direct reference signal you'll get for enterprise AI procurement — you should be setting caps, and tying them to compensation.

📍 Background

Coding agents blew past everyone's expectations. In the first four months of 2026, Uber spent its entire annual AI budget. This isn't unique to Uber — Claude Code, Codex, and Cursor burn tokens at a rate far beyond fixed subscriptions. Uber is the first major company to share hard numbers, and it's worth taking them seriously.

Why this matters

  • Quantitative benchmark: Uber caps each tool at $1,500/month/employee. Willison calculates ~$36k/year/engineer, which is 11% of US median SWE compensation ($330k). You can use this ratio to frame your own budget proposals.
  • Trend confirmation: AI tools are not free. Without caps, budgets burn out like Uber's. More companies will follow with similar policies soon.
  • Direct impact on you: If you're evaluating Codex or Claude Code enterprise procurement, this gives you a negotiating anchor. Also a reminder: personal subscriptions are cheap, but enterprise token consumption is orders of magnitude more expensive.
🎯 Action — what you can do today
  1. Read Simon Willison's full analysis (link below). He includes his own token usage data (~$1,000/month/provider) and a back-of-envelope calculation.
  2. Estimate your company's AI tool spend: without caps, a medium-intensity coding agent user could cost $2,000+/month. Use Uber's numbers as a baseline and draft a budget proposal.
  3. Watch for other giants: if Meta or Google announce similar limits, that's a 🚨 signal — the whole pricing model shifts.

What this means for clawii:

  • $1,500 reflects enterprise wholesale token pricing, not personal subscriptions. You won't spend $1,500/month on Claude Code as an individual (there's a cap), but enterprise deployments burn tokens at a completely different scale.
  • If your company is looking at bulk purchases, this article is the perfect negotiation piece — "Uber only gives $1,500, we shouldn't go higher."
  • Note: the cap is per tool, so an engineer using two tools gets $3,000/month total.

Honest caveat: Uber is a software company with high AI density. Non-tech teams may get lower caps. But the numbers themselves are a yardstick.

If other major companies (Meta, Goldman Sachs) mention similar caps or reduced AI spend expectations in quarterly calls, that's a 🚨. If Anthropic or OpenAI adjust enterprise pricing in response, also worth tracking.

👀 Worth a Look · Open-Source Security Framework

Anthropic open-sources AI-driven vulnerability discovery framework — #1 on HN with 438 points

Anthropic released an open-source framework on GitHub today that uses AI to automatically find security vulnerabilities in code. 438 points, #1 on HN, 122 comments. For clawii, this is the first major AI company to open-source an AI security tool — you can learn the approach to use in internal security reviews, or point your security team to evaluate it directly.

📍 Background

Anthropic has been moving aggressively on security: Project Glasswing (multi-party critical software security), the annual cyber threat report, and now a concrete tool. This framework plugs into CI/CD, uses Claude models to scan codebases for vulnerabilities, and suggests fixes. It's open-source under github.com/anthropics.

Why this matters

  • Open-source and auditable: you or your security team can inspect and deploy it directly, rather than relying on a closed-source security tool.
  • Major-company backing: from Anthropic, driven by Claude models — higher credibility than independent projects.
  • Lowers the security bar: traditional vulnerability scanning requires dedicated expertise and significant human effort. This AI framework accelerates discovery, especially for teams without dedicated security staff.
  • Relevant to you: even though you don't write code yourself, knowing this tool exists lets you raise it in security meetings or nudge your IT department to run a pilot.
🎯 Action — what you can do today
  1. Open the GitHub repo (link below), read the README to see what kinds of vulnerabilities it can find.
  2. Share with your security or IT team: ask them whether it can integrate into your existing CI/CD pipeline.
  3. Follow the HN discussion: 122 comments contain real-world feedback on accuracy, false positives, and limitations.

What this means for clawii:

  • You don't need to use it personally, but it marks the transition of AI security tools from concept to deployable product. As large enterprises adopt tools like this, compliance requirements and security procurement processes will change — you may need to mention this kind of tool in your annual security audit reports.
  • The framework runs on Claude, so you need an API key and token budget. If your company already has an Anthropic enterprise contract, deployment cost is nearly zero.

Honest caveat: AI-based vulnerability detection still has limitations — it's weaker on logic flaws and business logic issues. It's best at catching common patterns (XSS, SQL injection, path traversal) and cannot fully replace manual review.

🔭 What to watch for next

Two triggers: (1) if OpenAI or Google release similar competing tools, that signals a race in security AI frameworks — 🚨. (2) if this framework gets integrated into Claude Code or enterprise compliance tools, it dramatically lowers the barrier to adoption.
