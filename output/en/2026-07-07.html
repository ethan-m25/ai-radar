Tencent Hy3 open-source — 295B MoE model challenges GLM-5.2's open-source crown

  • Free until July 21 on OpenRouter — You can try it at zero cost and evaluate whether it fits your workflow.

Better models, worse tools?

Clawii — Simon Willison and Armin Ronacher both found a counterintuitive phenomenon: newer models (Opus 4.8, Sonnet 5) are actually worse at certain tool calls than older models. Not speculation — it's a training data bias. This directly affects your daily coding agent usage.

📍 Context · Armin Ronacher (Flask creator) reported in his blog "Better Models: Worse Tools" that Opus 4.8 and Sonnet 5 invent non-existent fields when calling Pi's edit tool, causing tool call failures. Simon Willison independently found similar issues. The likely cause: Anthropic trained newer models via RL to be better at Claude Code's built-in edit tools, but at the cost of compatibility with third-party tools.

Why this matters

  • Directly affects the coding agent you use — If you use non-Claude Code tools (Pi, custom agents), newer models might actually perform worse.
  • Reveals a side effect of RL training — The more a model is trained to excel at one specific tool, the worse it gets at others. The assumption "newer model = better" no longer holds.
  • Provides a practical workaround — Simon Willison shared a trick: tell Claude Code to delegate coding tasks to sub-agents running lower-power models, keeping the main model for judgment and review.
🎯 What you can do today
  1. Read Armin Ronacher's blog Better Models: Worse Tools for the technical details.
  2. If you use coding agents, test whether newer models (Opus 4.8, Sonnet 5) show similar issues with your tools.
  3. Try Simon Willison's trick: set up a sub-agent strategy in Claude Code where the main model uses its judgment to decide when to call lower-power models.
