GPT-5.6 officially launches — three tiers, GA

Hey clawii — OpenAI today released the GPT-5.6 family, including Sol (flagship), Terra (mid-tier), and Luna (low-cost). Sam Altman calls Sol "the best model we have ever produced." Pricing ranges from $1 to $5 per million tokens, with a 90% cache read discount. Simon Willison's blog has a detailed review and comparison.

📍 Context · GPT-5.6 was previewed on June 26 under a restricted U.S. government rollout limited to trusted partners. Today's release means general availability across ChatGPT, Codex, and the API. New features include ChatGPT Work (a desktop app merging Codex and ChatGPT), Sites beta, programmatic tool calling, and multi-agent beta. Pricing: Sol $5/M input, $25/M output; Terra $2/M input, $10/M output; Luna $1/M input, $5/M output. 90% cache read discount.

Why this matters

  • General availability means you can use it right now — the preview required an application; now it's accessible via API and ChatGPT. If you use Codex, Sol is the default model.
  • Competitive pricing — Sol is cheaper than Opus 4.8 ($5/$25 vs $15/$75), Terra matches Sonnet 5 ($2/$10), and Luna is the cheapest flagship-tier model available.
  • Simon Willison's review shows Sol close to Fable 5 on coding agent tasks — Intelligence Index score of 59, slightly below Fable 5 but at one-third the cost.

  1. Switch your API model to gpt-5.6-sol or gpt-5.6-terra on OpenAI API and compare coding performance.
  2. If you use Codex, it should already default to Sol. Try running a task you previously did with Sonnet 5 and see how the quality compares.
  3. Run the numbers: if you consume 10M input + 2M output daily, Sol costs ~$100/day, Terra ~$40, Luna ~$20. Compared to Opus 4.8 at ~$300, that's a significant saving.

Honest take: Simon Willison's review gives Sol a 59 on the Intelligence Index for coding agents, with Fable 5 scoring higher but at 3x the cost. Terra and Luna are value picks. But note Sol's context is only 128K — much smaller than Sonnet 5's 1M — so it's not ideal for large codebases.

🔭 What to watch for · Independent coding benchmark results (SWE-bench, Code Arena) for GPT-5.6 Sol. Community comparisons between Sol and Fable 5 on real coding tasks. The impact of cache discounts on actual cost. If Sol approaches Fable 5 performance on coding agent workloads, OpenAI's pricing strategy could reshape coding agent economics.
Jargon decoder · Token = the smallest unit a model processes; ~1 English word ≈ 1-2 tokens, ~1 Chinese character ≈ 2-3 tokens. Cache read = when the same input is reused, the model can read cached results, dramatically reducing cost. GPT-5.6's 90% discount means cache hits cost only 10% of the original price.

Muse Spark 1.1 — Meta opens API, improved tool calling

Hey clawii — Meta released Muse Spark 1.1, the first Spark model with an API. Simon Willison notes significant improvements in agentic tool calling and computer use. He built an LLM plugin so you can use the model from the command line.

📍 Context · Muse Spark is a lightweight model Meta released in April, focused on efficient inference. Version 1.1 opens the API for the first time and improves tool calling and computer use capabilities. Meta also published a detailed evaluation report, including the interesting "attractor states in self-conversation" phenomenon.

Why this matters

  • Another usable coding agent model — while Muse Spark isn't specifically designed for coding, the tool calling improvements mean it can be used for agent tasks. Worth a try if you're in the Meta ecosystem.
  • Simon Willison made an LLM plugin — you can get started fast with uv tool install llm and llm install llm-meta-ai.
  • Pricing unknown — API pricing hasn't been announced yet, but it's expected to be cost-competitive.
