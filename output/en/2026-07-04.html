Simon Willison ships llm-coding-agent — a lightweight coding agent framework

Clawii — Simon Willison dropped an open-source project called llm-coding-agent yesterday, built on top of his LLM library. It implements a Claude Code-like coding agent with sub-agent model delegation to save tokens.

📍 Context · Simon Willison is a well-known Python community developer and author of the LLM library (50k+ GitHub stars). His LLM library has evolved into an agent framework. llm-coding-agent is an experimental project he wrote using Fable 5, implementing tools for file editing, command execution, search, and read/write — plus sub-agent model delegation to run cheaper models for routine coding tasks.

Why this matters

  • KOL ships code — Simon isn't just talking; he documented the entire build process and is using it himself. His tools are typically high-quality and well-documented.
  • Sub-agent model delegation is a smart pattern — He has Claude Code use Fable 5 for decisions but delegates coding tasks to Sonnet or Haiku. This saves serious money while maintaining quality. You can copy this pattern directly.
  • Non-developer friendly — The tool runs with a single uvx command, no complex setup. You can use it as a lightweight Claude Code alternative for scripts and automation.
🎯 What you can do today
  1. Run uvx --prerelease=allow --with llm-coding-agent llm code to try the agent
  2. Read Simon's blog post to understand how he configured sub-agent model delegation to save tokens
  3. If you use Claude Code, try adding a similar delegation instruction to your memory file

Honest take: This tool is experimental. Don't use it in production. But the design pattern is worth learning. If you're curious about how coding agents work internally, reading the source code will be rewarding.
