Claude Tag — your AI teammate now lives inside Slack

Hey clawii — Anthropic launched a native Slack integration called Claude Tag yesterday (June 23). This isn't another chatbot. It's an AI team member that lives in your Slack channels, receives async tasks by @-mention, and writes actual code and PRs. Anthropic's own product team already uses it for 65% of their code and PRs.

📍 Context

Claude Code (the terminal IDE agent) is powerful but fundamentally single-player — you in your terminal. Claude Tag is the next step: putting Claude into the center of team collaboration. It's designed as an "async, multiplayer, proactive" workflow layer, not a simple Q&A bot. You @-mention it with a task and it pushes code back into the channel.

Why this matters

  • Team AI enters Slack — If you already use Claude Code, Tag is an extension, not a replacement: it reviews PRs while you sleep, runs tests while you're in meetings.
  • For a non-developer (clawii) — This might sound distant, but the trend is clear: AI becomes a colleague, not just a tool. If your company adopts Claude Enterprise, you could eventually @Claude in a Slack channel to generate an Excel report or fix a formula.
  • Anthropic's own numbers — 65% of code and PRs written via Tag. That's higher than the 80%+ merged-code stat they reported from Claude Code alone. Embedding into the collaboration layer is more frequent than single-terminal usage.
  • Currently beta — Claude Enterprise and Team plans only. Pricing not separately announced.
🎯 Action — what you can do today
  1. If you're on a Claude Enterprise plan, contact Anthropic to request beta access for Claude Tag.
  2. Understand the usage pattern: @Claude in a channel with a task. It replies in-channel, visible to the whole team.
  3. Watch for the broader trend — expect more Slack-native AI agent products in the coming months.

How it works — Under the hood it uses Claude Fable 5 / Opus 4.8 models (configurable). All inference is done via Anthropic's backend API — no local model. Admins grant channel and tool permissions. Users just @Claude.

How hard to set up — No installation for end users. Admins add the Slack app and authorize permissions. A team can be up in minutes.

What you can actually use it for — Code review, PR generation, issue triage, channel summarization, test execution, deployment commands. For non-developer teams: eventually, data reports, spreadsheet edits, and document generation in channel.

Honest caveat — Beta only, features may be unstable. Currently aimed at engineering teams. HR or finance probably can't use it yet. But it's a clear signal: AI embedding into team workflows is inevitable.

🔭 What to watch for next

Public launch (out of beta). Significant adoption case studies from non-engineering teams. Or competitive responses from OpenAI/Microsoft for Slack AI integration. I'll keep tracking.

VibeThinker-3B — Weibo AI open-sources a tiny 3B reasoning model you can run locally

Weibo AI (Weibo) open-sourced VibeThinker-3B last week. It's a 3B-parameter model that hit HuggingFace trending at #9 with 41k downloads in 5 days. At 3B params, it'll run on any modern laptop or M‑series Mac via Ollama or llama.cpp. No international KOL buzz (yet), but the small-model trend keeps accelerating.

Why it's worth knowing the name

  • 3B is the local sweet spot — With GGUF quantization it's ~1.5GB. Any MacBook or GPU with 4GB VRAM can run it.
  • From Weibo AI (Chinese social media giant) — Named "VibeThinker," suggesting optimization for reasoning/thinking tasks.
  • 41k downloads in 5 days — Strong Chinese community interest. No English KOL discussion yet, but when it hits Red Note or WeChat, you'll already know the name.
  • Part of a broader small-model wave — Qwen 3.6 27B, Gemma 4 12B, Mistral Small 4. VibeThinker enters this arena at the smallest end.
