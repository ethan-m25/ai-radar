Grok 4.5 launches — coding agent price war begins

Hey clawii — xAI dropped Grok 4.5 today, a 1.5 trillion parameter coding agent model priced at $2/$6 per million tokens — at least 5x cheaper than Opus 4.8. 📍 Context · The coding agent model market is splitting fast: OpenAI has GPT-5.5/5.6 Codex, Anthropic has Opus 4.8/Sonnet 5, open-source has GLM-5.2/Ornith. xAI's Grok was previously focused on chat; v4.5 is their first model aimed squarely at coding and agents, emphasizing capability-per-dollar over raw benchmarks.   • 1M context window — Returning to 1M tokens soon, good for large codebases.
  • Ecosystem support — Already available via Grok Build/API, Hermes Agent, Portal, and OpenRouter. Seamless swap-in for your current models.

OpenAI GPT-Live voice mode upgrade

Hey clawii — OpenAI upgraded ChatGPT's voice mode today. The new GPT-Live model is dramatically better, and it can delegate complex tasks to GPT-5.5 in the background while you keep talking. Simon Willison tested it for an hour and calls it "the most useful voice mode yet." You mostly vibe-code, but for daily queries and brainstorming, voice can be faster than typing.

📍 Context · ChatGPT's old voice mode was based on a GPT-4o-era model with a knowledge cutoff in 2024 — limited in scope and usefulness. GPT-Live uses a much newer model, supports real-time conversation, interruptions, emotion detection, and hands off hard tasks (search, reasoning) to GPT-5.5 asynchronously while keeping the conversation flowing. Simon Willison says he "mostly stopped using voice mode before this; now I might use it all the time."

Why this matters

  • Massive quality jump — No longer a "dumb voice assistant." It understands complex questions and can even discuss programming ideas with you.
  • Background delegation mechanism — Voice mode can trigger GPT-5.5 for search and reasoning, with results coming back asynchronously without interrupting the flow. It's like a primitive multi-agent setup.
  • Useful in your daily life — Walking, driving, doing chores — you can talk through coding problems and the model remembers context across sessions.
🎯 What you can do today
  1. Open the ChatGPT mobile app, enable voice mode, and try GPT-Live.
  2. Discuss a programming problem out loud — see if it understands and gives useful advice.
  3. Notice when it silently delegates a hard task to the background model — the conversation shouldn't stutter.

Bun rewrite case study — agent engineering at its limit

Hey clawii — Simon Willison published a detailed breakdown of the full Bun rewrite from Zig to Rust, carried out with Claude Code / Fable. Cost: $165k. Tokens: 5.9B input, 690M output. Not a tool you can use directly, but it shows what agents can do at the edge: rewrite a production runtime in 11 days with near-perfect test suite pass rates.

📍 Context · Bun is a JavaScript runtime (like Node.js), originally written in Zig. Creator Jarred Sumner decided to rewrite it in Rust using Claude Code, driven by Zig's memory management issues causing a long tail of bugs. The rewrite leveraged Bun's TypeScript test suite as a conformance suite — agents automatically translated code and an adversarial review loop caught regressions. The final PR was +1M lines. When it shipped, almost nobody noticed — except startup got 10% faster on Linux.

Why this matters

  • A new dimension of agent capability — This isn't code completion. It's a complete language-level rewrite involving massive logic translation and edge-case coverage.
  • The test suite is the linchpin — A language-independent conformance suite let the agent auto-verify correctness. That's the foundation for any large-scale agent project.
  • The cost barrier — $165k in API costs (internal pricing was lower) means this scale of agent work still needs serious budget. But that threshold is dropping fast.
