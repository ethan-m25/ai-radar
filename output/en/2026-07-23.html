POCKET: 35B-parameter model runs on iPhone

Hey clawii — there's an interesting HuggingFace community article today about POCKET, a 35B-parameter model that claims to run on an iPhone, and even on a PC with no GPU. If you've been waiting for a powerful model that runs locally on your phone, this might be the answer.

📍 Context · Local AI inference has always been limited by model size and hardware. Tools like LM Studio, MLX, and Nativ make it possible to run models on Mac and PC, but mobile devices have lacked large-parameter models. POCKET, released by the FINAL-Bench team, claims 35B parameters running on iPhone without a GPU. Is this achieved through extreme quantization and architecture optimization?   • Open-source potential — The model is published on HuggingFace, likely open-weight, so you could self-host or fine-tune it.

GigaToken: Claims 1000x language model tokenization speedup

Hey clawii — there's a project on HN today called GigaToken that claims to be 1000x faster than conventional tokenization. Since you use AI for coding all the time, tokenization happens on every request — faster tokenization means quicker responses and lower costs.

📍 Context · Tokenization is the process of splitting text into tokens that a model can process. It's the first step in the LLM inference pipeline, typically using BPE or WordPiece algorithms. GigaToken is a new open-source library claiming 1000x speedup through optimized algorithms and parallelization. It hit #5 on HN with 521 points and 110 comments, showing strong community interest.

Why this matters

  • Direct inference speed boost — Tokenization is one of the bottlenecks in LLM inference. A 1000x speedup means time-to-first-token (TTFT) drops dramatically, which matters for real-time interactive scenarios.
  • Lower API costs — Faster tokenization means fewer compute resources consumed, potentially reducing usage costs.
  • Open source — The project is on GitHub, so you could integrate it into your own toolchain, or benefit if it gets adopted by tools like Codex or Claude Code.
🎯 What you can do today
  1. Check the GitHub repo: github.com/marcelroed/GigaToken
  2. Read the README to understand the speedup mechanism and how to use it.
  3. If you run models locally, try swapping in the GigaToken tokenizer to see if it's compatible.

Cactus Hybrid: Teaches Gemma 4 to know when it's wrong

Hey clawii — there's a Show HN project called Cactus Hybrid that teaches Gemma 4 to recognize when it might be wrong. Not a huge score, but the direction is interesting — making models more reliable.

📍 Context · Model calibration is about making AI aware of its own uncertainty. Cactus Hybrid fine-tunes Gemma 4 or adds a confidence prediction head so the model can signal "I don't know" alongside its output. This can reduce hallucination risk.

Why this matters

  • Reduces hallucinations — If the model knows it's uncertain, it can tell you "I'm not sure" instead of making something up.
  • Open-source project — On GitHub, you could try integrating it into your own agent.
  • Trend signal — Calibration is becoming a key component of agent safety.
