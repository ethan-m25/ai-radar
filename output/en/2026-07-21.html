Nativ: run frontier open models locally on your Mac

Hey clawii — a new project called Nativ hit HN front page today. It lets you run frontier models like Claude Fable 5 and GPT-5.6 Sol locally on your Mac. Built on llama.cpp and MLX, no GPU required — runs smoothly on M-series chips. For you, this means testing and using frontier models locally without cloud API costs, with better privacy.

📍 Context · Running large models locally has always been a challenge, but Nativ uses advanced quantization to make hundreds-of-billions-parameter models feasible on a Mac. It supports LLaMA, Mistral, Qwen, and more, leveraging Apple's MLX framework for optimized inference on M1/M2/M3/M4 chips. For developers who want to experience frontier models locally without wrestling with the command line, this is a great entry point.

Why this matters

  • Local models, better privacy — All data stays on your machine, never uploaded to the cloud. Huge advantage for sensitive projects or experiments.
  • Lower cost — No API fees for local models. Great for frequent testing and iteration.
  • Lowers the barrier to frontier models — No GPU needed, just a Mac. Run models comparable to Claude and GPT level locally.
🎯 What you can do today
  1. Visit Nativ's GitHub repo: github.com/blaizzy/nativ, follow the README to install.
  2. Download a model (e.g. Qwen 3.6 27B or Mistral Small 3.1) and run it with Nativ.
  3. Compare against your vibe coding workflow — see if local models can replace cloud API for some tasks.

stencil.so: one single edit, frontier model handles the rest

Hey clawii — a new tool called stencil.so hit HN today. It claims you only need one edit for the frontier model to do the work. It uses smart caching and prompt management to reduce repetitive input and boost efficiency. For someone using Claude Code daily, this could mean less waiting and a smoother workflow.

📍 Context · When using AI coding agents, you often repeat the same or similar instructions, wasting time and tokens. stencil.so solves this with a "template" mechanism — save common prompts and reuse them with one click. It supports variables and conditional logic for flexible templates. For anyone who uses AI coding agents frequently, this can significantly cut down repetitive work.

Why this matters

  • Saves time — No more manually typing the same prompts. One-click template invocation.
  • Reduces token consumption — Reuse templates instead of regenerating the same prompt content, saving API costs.
  • Improves workflow consistency — Templates ensure consistent prompt quality every time, avoiding missed key information.
🎯 What you can do today
  1. Visit stencil.so, sign up and create your first template.
  2. Import your common Claude Code prompts — e.g. "generate React component", "debug Python code".
  3. Use stencil templates during coding and feel the efficiency boost.
