Simon Willison 深度体验：Claude Fable “relentlessly proactive”——主动到有点过

昨天雷达报道了 Fable 5 的静默干预回滚，但真正值得你周末花时间看的，是 Simon Willison 今天发的 2400 词的体验报告。他给 Fable 5 下的定义是：“ relentlessly proactive”——模型知道你下一步可能要做什么，会直接帮你做完，甚至在你要求之前。这种“过度主动”既是超能力也是坑。

📍 背景

Simon Willison 是 Python 社区知名人物，llm 命令行工具的作者。他是最早拿到 Fable 5 访问权限的非 Anthropic 员工之一。过去两天他高强度使用 Fable 5 完成了多项任务：重写 llm 工具的 ask_user 功能、构建 RAG 原型、做 prompt injection 测试。他的体验报告是目前最详实的第三方使用反馈。Fable 5 的价格是 $10/百万输入 token，$50/百万输出 token，比 Opus 4.8 贵很多。

为什么这事重要

  • “主动”是 Fable 5 最核心的差异化 — Simon 说这和 Opus 4.8 完全不同：Fable 5 会主动提出优化建议、调用工具、创建新文件，而不是等待你的指令。它像是一个过于热心的实习生，在你说“能帮我查一下这个问题吗”之前就已经查完了。
  • 价格是最大的拦路虎 — Simon 估算一篇 2400 词的博客文章（包括所有实验）花了 $150 的 API 费用。他开玩笑说“幸好 Anthropic 给我免费用，否则我不会这样用”。对自费用户来说，Fable 5 的成本是 Opus 4.8 的 5-10 倍。
  • 对 vibe coding 有实际参考价值 — Simon 用 Fable 5 重写了 llm 工具的 ask_user 功能（允许 AI 工具在执行过程中向用户提问），这个过程只花了 90 分钟。他评价说“如果这是未来工作的价格，那值得。但现在太贵了。”
  • HN 热度持续 — 今天 HN 上“Claude Fable is relentlessly proactive”排 #11（477 分），“Anthropic apologizes for invisible guardrails”排 #12（433 分），MiMo Code 开源排 #13（504 分）。Fable 5 仍然是社区焦点。
🎯 今天就能做的事
  1. 读 Simon 的完整博客 — “Claude Fable is relentlessly proactive”。2400 词，但值得读完。
  2. 亲自体验 Fable 5 — 如果你有 Anthropic API 访问权限，尝试用 Fable 5 完成一个真实的办公/代码任务。注意观察它是否主动采取额外行动。
  3. 对比成本 — 打开你的 API 使用记录，看看 Fable 5 和 Opus 4.8 的成本差异。对于日常任务，Opus 4.8 可能更合适。
  4. 在 Claude Code 中试用 — 如果你用 Claude Code，尝试设置 --model claude-fable-5，看看对 coding 效率的影响。

为什么 Fable 5 更主动？ — Simon 分析说 Fable 5 在系统提示中内置了“主动行为”模式。它不仅仅是回答你的问题，而是会：主动建议更好的实现方案、自动运行代码检查错误、在发现 bug 时直接修复而不等你要求、在完成一个任务后主动提出下一个步骤。这在需要快速迭代的场景下非常强大，但在需要精确控制输出时可能带来麻烦。

Fable 5 适合做什么？ — Simon 发现它特别适合：1）需要多个工具链调用的复杂任务；2）需要快速原型的项目；3）需要“灵感”的开放性问题。不适合：1）成本敏感的大量批次任务；2）需要高度可重复输出的任务；3）初学者使用（因为过度主动可能导致混乱）。

对 clawii 的影响 — 作为补偿经理做 vibe coding，Fable 5 可以用在你需要“创意突破”的任务上：比如从零开始搭建一个自动化脚本、处理复杂的 Excel 数据管道、或者需要探索性分析的场景。日常办公任务（写邮件、做报告）用 Opus 4.8 完全够用，还便宜很多。建议的策略是：Fable 5 作为“灵感引擎”，Opus 4.8 作为“执行引擎”。

诚实说缺点 — 1）成本高到离谱（Simon 说“这限制了我愿意用它做的事”）；2）过度主动可能导致安全风险（容易被提示注入利用）；3）模型会占用很多 token 用于非核心任务（如主动搜索、分析代码库结构），导致成本进一步上升；4）没有公开的量化性能数据，只有 Anthropic 自己的基准。

🔭 后续关注什么

Relentlessly proactive
极度主动——模型不仅回答问题，还会主动采取行动、提出建议、甚至执行额外任务。
ask_user
一个 AI 工具功能，允许模型在执行过程中向用户提问以获取更多信息，而不是在不确定时胡乱猜测。

攻击者通过巧妙构造输入来欺骗模型执行意外操作。Fable 5 的过度主动增加了被注入的风险。
