代码整洁度影响编码 agent 成功率

clawii，一篇对照研究证明：代码整洁度直接决定 agent 能否成功完成任务。不是玄学，是实验数据。HN 139 点，75 条评论。

📍 背景 · 一篇来自 arXiv 的论文《Does code cleanliness affect coding agents? A controlled minimal-pair study》通过控制变量实验，测试了代码整洁度对编码 agent（如 Codex、Claude Code）成功率的影响。结果发现，整洁的代码库可以让 agent 的任务成功率提高 20-40%。这个发现对 vibe‑coding 的实践有直接指导意义。

为什么这事重要

  • 直接影响你的 vibe‑coding 效率 — 如果你经常让 agent 修改现有代码，代码的整洁度（命名、注释、模块化）会直接影响 agent 的理解能力和修改质量。
  • 提供了可操作的改进方向 — 与其抱怨 agent 不够聪明，不如先检查你的代码库是否整洁。论文给出了具体的整洁度指标。
  • 对团队协作有指导意义 — 如果你在团队中推广 AI 辅助编码，代码整洁度应该成为代码审查的一部分。
🎯 今天就能做的事
  1. 阅读论文摘要 Does code cleanliness affect coding agents?，了解具体的实验设计和结论。
  2. 检查你当前项目的代码整洁度：变量命名是否一致？函数是否过长？是否有足够的注释？
  3. 尝试在一个整洁的代码库和一个混乱的代码库上运行相同的 agent 任务，对比成功率和输出质量。

对你最相关的：如果你经常让 agent 修改代码，花 10 分钟重构混乱的代码库可能比花 30 分钟调试 agent 的输出更有效。

老实说：这个研究的样本量有限（约 100 个任务），但结论和直觉一致。代码整洁度对 agent 的影响可能因模型而异——更强的模型（如 Sol Ultra）可能对混乱代码的容忍度更高。

🔭 关注点 · 是否有更大规模的研究验证这个结论？编码 agent 工具（如 Codex、Claude Code）是否会内置代码整洁度检查？如果结论被广泛接受，可能会改变团队代码审查的标准。
