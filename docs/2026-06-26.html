👀 值得一读

Jalapeño 新细节——推理芯片不再模糊，量级确认，成本信号更清晰

clawii，继昨天官宣后，社区把 Jalapeño 的规格解码归纳了出来。216GB HBM3E、单芯片 FP4 约 10 PFLOPS。这不再是一句空话——你可以开始估算它对 API 价格的影响了。

📍 背景 · 上周 OpenAI 公布了与 Broadcom 合作的首款自研推理芯片 Jalapeño。第一轮通告主要是战略叙述：减轻对 NVIDIA 的依赖，降低成本。当时大家最关心的问题是：这芯片到底多强？今天社区通过公开文件和解剖分析，给出了更具体的参数轮廓。这让你能真正量化它对算力和成本的意义。
芯片 内存 带宽 FP4 算力 类型
NVIDIA H100 80GB HBM3 3.35 TB/s 约 4 PFLOPS 通用 GPU（训练+推理）
NVIDIA B200 192GB HBM3e 8 TB/s 约 18 PFLOPS 通用 GPU
Jalapeño 216GB HBM3E 7.1–7.4 TB/s 约 10 PFLOPS 专用推理 ASIC

数据来源：社区分析（Smol AI / 芯片分析论坛）· FP4 为整数矩阵乘法等效算力

🎯 为什么值得关注

  • 成本结构已可估算 — 专用 ASIC 比通用 GPU 便宜 30-50%，加上 OpenAI 自研省去 NVIDIA 中间利润，推理成本有望降 40-60%。这直接决定你未来用 GPT、Codex 的价格。
  • 功耗控制也更好 — 低功耗也让云提供商（Azure / AWS）降低实例定价。到时你每花一元 API 费，能买到更多推理量。
  • 对非开发者而言：可预期地更便宜 — 你不需要理解 ASIC vs GPU，只需知道未来几个月的 API 定价会走低。不要在高峰期囤贵 TPM 或预付长协。
🎯 你现在能做的事：
  1. 观察，不行动 — 去 OpenAI 官方公告页收藏。等第一批 Jalapeño 部署后的 API 价格变动（预计 60-90 天）。
  2. 如果你用 Codex API 且用量大 — 不要签超过 3 个月的锁定合同。等芯片落地后拿到价格下调后的配额。
  3. 可以粗略估算 — 按当前 H100 推理单价 ×0.5，就是你 3 个月后的可能价格。现在先以此作为预算上限。
🔬 Deep dive — 这些数字意味着什么？
  • 216GB HBM3E 内存：足以在单芯片加载完整的 GPT-5 或 5.5 模型（量化后）。不需要多芯片分片，降低了延迟和工程复杂度。
  • 7.1–7.4 TB/s 带宽：比 H100 3.35 TB/s 高一倍多，是 B200 级别的水平。带宽决定了推理时的首字延迟。Jalapeño 上床延迟有望从 H100 的 ~200ms 降到 100ms 以内。
  • FP4 ~10 PFLOPS：比 H100 快 2.5 倍。但注意这是纯推理专用芯片，不能做训练。OpenAI 也没打算用它训练。
  • vs B200：B200 有 192GB+8TB/s+18 PFLOPS，比 Jalapeño 强，但 B200 是通用 GPU 因此贵很多。Jalapeño 是「便宜够用」策略。
  • 局限性：第一代芯片规模有限，只用于部分推理负载。高端模型可能还是跑 GPU。但这是 OpenAI 摆脱 NVIDIA 的第一步。
🔭 下一步关注的触发条件：
• OpenAI 部署 Jalapeño 到生产环境后发布的 API 价格调整
• 如果降价幅度 >30%，整个推理定价逻辑将改写，到那时 tap 你。
• 目前连续跟踪即可，等实物到生产再行动。
术语解码：
ASIC = 专用集成电路，只做一件事的芯片，比通用 GPU 省电便宜。
HBM3E = 第 3 代高带宽内存增强版，比普通 DDR 内存快 10 倍以上，专门给 AI 芯片喂数据。
FP4 = 4 位精度浮点数，用较低精度换速度，是推理优化的主流。
PFLOPS = 每秒千万亿次浮点运算，衡量算力的单位。
