模型越强，工具调用越差？— 持续发酵

clawii，昨天 Simon Willison 和 Armin Ronacher 发现的"新模型在某些工具调用上反而比旧模型差"的话题今天在 HN 继续发酵。社区分享了更多案例，包括 GPT-5.5 和 Gemini 3.5 Flash 也有类似问题。不是玄学，是训练数据偏差。这对你日常使用编码 agent 有直接影响。

📍 背景 · Armin Ronacher（Flask 作者）在博客《Better Models: Worse Tools》中报告：Opus 4.8 和 Sonnet 5 在调用 Pi 的编辑工具时，会凭空发明不存在的字段，导致工具调用失败。Simon Willison 也独立发现了类似问题。今天社区补充了更多案例，包括 GPT-5.5 和 Gemini 3.5 Flash 也有类似行为。原因可能是 RL 训练过度优化了特定工具，牺牲了对第三方工具的兼容性。

为什么这事重要

  • 直接影响你使用的编码 agent — 如果你在用非 Claude Code 的工具（如 Pi、自定义 agent），新模型可能反而表现更差。
  • 揭示了 RL 训练的副作用 — 模型被训练得越擅长某个特定工具，就越不擅长其他工具。这意味着"模型越新越好"的假设不成立。
  • 提供了实用的应对策略 — Simon Willison 分享了一个技巧：让 Claude Code 用子 agent 调用低功耗模型处理编码任务，保留主模型做判断和审查。
🎯 今天就能做的事
  1. 阅读 Armin Ronacher 的博客 Better Models: Worse Tools，了解具体的技术细节。
  2. 如果你在用编码 agent，测试一下新模型（Opus 4.8、Sonnet 5、GPT-5.5）在你常用的工具上是否出现类似问题。
  3. 尝试 Simon Willison 的技巧：在 Claude Code 中设置子 agent 策略，让主模型用判断力决定何时调用低功耗模型。

对你最相关的：如果你主要用 Claude Code，新模型可能更好。如果你用其他编码 agent 工具，可能需要暂时锁定旧模型。

老实说：这个问题可能只影响特定工具调用场景。对于简单的聊天和代码生成，新模型仍然更好。但如果你依赖工具调用的精确性，需要谨慎升级。

小模型在不稳定网络地区崛起

clawii，IEEE 报道了一个趋势：小模型（<7B 参数）在不稳定网络地区开始被采用。不是新工具，但反映了行业的一个重要转向——不是所有人都需要旗舰模型。

📍 背景 · IEEE 的文章《Small AI Models Gain Traction In Places with Unreliable Networks》指出，在非洲、东南亚等网络不稳定的地区，小模型（如 Gemma 4、Qwen 3.5 的小版本）正在被用于离线翻译、农业咨询、医疗诊断等场景。这些模型可以在低端手机上运行，不需要云端连接。

为什么这事重要

  • 反映了行业的重要转向 — 不是所有人都需要旗舰模型。小模型在特定场景下可能更实用、更经济。
  • 对你也有参考价值 — 如果你在出差、旅行或网络不稳定的环境中工作，小模型可能是更好的选择。
  • 可能影响你的工具选择 — 如果小模型被集成到主流工具中，你可能会有更多离线可用的选项。
