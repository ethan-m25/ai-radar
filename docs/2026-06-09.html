Apple Siri AI 基于 Gemini 模型，跑在 Google Cloud + NVIDIA 上

Apple 在 WWDC 2026 上宣布 Siri AI 基于 Google Gemini 模型，跑在 Google Cloud 的 NVIDIA GPU 上，611 分登顶 HN。这意味着 Apple 放弃了自研大模型路线，转而使用第三方模型，对 AI 行业格局有重大影响。

📍 背景

Apple 在 2024 年 WWDC 上宣布 Apple Intelligence，但实际落地效果不佳。今年 WWDC 上，Apple 承认自研模型能力不足，转而与 Google 合作，使用 Gemini 模型作为 Siri AI 的底层引擎。模型跑在 Google Cloud 的 NVIDIA GPU 上，通过 Private Cloud Compute 架构保护隐私。

为什么这事重要

  • Apple 放弃自研大模型 — 这是 Apple 首次公开承认自研 AI 模型不如第三方，转而使用 Google 的模型。对 Google 来说是重大胜利，对 OpenAI 和 Anthropic 是警讯。
  • 对 clawii 的实际影响 — 如果你用 iPhone，Siri AI 将变得更强大。但更重要的是，这验证了 Gemini 模型的能力——如果你在考虑切换模型，Gemini 现在有了 Apple 的背书。
  • 隐私架构值得关注 — Apple 的 Private Cloud Compute 扩展到 Google Cloud，使用 NVIDIA GPU，但保持了相同的安全承诺。这意味着企业级 AI 部署可以在第三方云上实现隐私保护。
🎯 今天就能做的事
  1. 读 Apple 安全研究博客 — 去 security.apple.com 看 "Expanding Private Cloud Compute" 文章，了解他们如何在 Google Cloud 上保持隐私。
  2. 试 Gemini 模型 — 在 Google AI Studio 或通过 API 试用 Gemini 3.5 或 Gemini Omni，对比它和 GPT-5.5、Claude Opus 4.8 的效果。
  3. 关注 Siri AI 评测 — 等 iOS 27 开发者 beta 的评测出来，看看 Siri AI 实际表现如何。如果效果好，说明 Gemini 在 agentic 任务上很强。
  4. 思考企业部署 — 如果你在公司做 AI 采购，Apple 的 PCC 架构是一个很好的参考：如何在第三方云上安全部署 AI。

跑在什么之上 — Siri AI 使用 Google Gemini 的定制版本，通过 Google Cloud 的 NVIDIA GPU 集群推理。Apple 的 Core AI Framework 提供了 PyTorch 集成，开发者可以用 coreai-torch 包将 PyTorch 模型部署到 Apple 硬件上。

上手难度 — 对普通用户：等 iOS 27 更新即可。对开发者：Core AI Framework 需要 Xcode 和 iOS 27 SDK，学习曲线中等。PyTorch 集成相对简单，如果你已经会用 PyTorch。

诚实说缺点 — Apple 没有完全放弃自研，只是承认在通用大模型上不如 Google。Siri AI 目前只支持英文，中文支持时间未公布。PCC 扩展到 Google Cloud 增加了供应链风险。

你能拿它做什么 — 如果你用 iPhone，Siri AI 将能帮你做更复杂的事情：预订餐厅、管理日程、分析文档。如果你做 AI 开发，Core AI Framework 让你能在 Apple 硬件上高效运行模型。

小米 MiMo-v2.5-Pro-UltraSpeed — 1T 模型，1000 token/s

小米发布 MiMo-v2.5-Pro-UltraSpeed，一个 1T 参数的 MoE 模型，推理速度达到 1000 token/s，580 分登上 HN。这是中国厂商在推理速度上的重大突破，可能改变 API 定价格局。

📍 背景

小米的 MiMo 系列是他们的 MoE 模型家族。v2.5-Pro-UltraSpeed 是速度优化版本，通过架构创新（可能是更激进的 MoE 路由 + 量化 + 推测解码）实现了 1000 token/s 的推理速度。1T 总参数，但每次只激活一小部分（估计 50-100B），所以速度很快。

为什么这事重要

  • 速度是新的护城河 — 当模型精度接近天花板时，速度成为关键差异化因素。1000 token/s 意味着实时对话、快速代码补全、大规模批处理都成为可能。
  • 对 clawii 的实际影响 — 如果你用 API 做 vibe coding，速度直接决定你的工作流效率。1000 token/s 比 GPT-5.5 快 5-10 倍，成本可能更低。
  • 中国厂商的竞争力 — 小米、DeepSeek、阿里等中国厂商在推理优化上持续突破，可能在未来 6 个月内改变全球 API 定价格局。
🎯 今天就能做的事
  1. 读小米官方公告 — 去 xiaomi.com 看 MiMo-v2.5-Pro-UltraSpeed 的技术细节，了解他们用了什么优化技术。
  2. 试 API — 如果小米开放了 API（通过 OpenRouter 或直接），注册并测试速度。用 curl 测一下 token 生成速度。
  3. 对比成本 — 计算 1000 token/s 的 API 价格 vs GPT-5.5 的价格。如果便宜 10 倍，考虑切换后端模型。
  4. 关注第三方评测 — 去 HN 帖子下面看有没有人做独立速度测试。如果社区验证通过，这个信号会升级。

上手难度 — 用 API 最简单：注册、充值、调用，5 分钟。自托管不可能（1T 参数需要大量 GPU）。通过 OpenRouter 可以零配置使用。

诚实说缺点 — 速度数据来自小米官方，尚未被独立验证。精度可能不如 GPT-5.5 Pro 在某些任务上。API 稳定性未知（中国厂商的 API 有时不稳定）。中文能力很强，英文可能略逊。

你能拿它做什么 — 如果你做实时对话应用、代码补全、或需要高吞吐量的批处理任务，MiMo-v2.5 的速度优势非常明显。对于 vibe coding，更快的响应意味着更流畅的交互。

MoE (Mixture-of-Experts)
混合专家模型，把一个大模型拆成多个"专家"子网络，每次只激活一部分，从而在保持能力的同时降低计算成本。
推测解码 (Speculative Decoding)
一种加速推理的技术，用一个小的"草稿"模型先生成多个 token，再用大模型验证，从而并行生成多个 token。

OpenEnv — 开源社区力挺的 agentic RL 标准环境

HuggingFace 博客宣布 OpenEnv 获得开源社区 +13 票支持，成为 agentic RL（强化学习）的标准测试环境。这对 agent 开发者和企业部署 agent 系统有重要意义。

📍 背景

Agentic RL 是让 AI agent 通过试错学习如何完成任务的训练方法。OpenEnv 是 Meta 开源的 agent 评估环境，类似于 OpenAI Gym 但专门为 agent 设计。它提供标准化的任务和评估指标，让不同 agent 系统可以公平比较。社区投票支持它成为标准，意味着它可能成为 agent 开发的"Linux"。

为什么这事重要

  • 标准化是生态成熟的标志 — 当 agent 开发有了标准测试环境，不同团队可以公平比较，加速进步。就像 ImageNet 对计算机视觉的意义。
  • 对 clawii 的实际影响 — 虽然你不直接做 RL，但标准环境意味着 agent 工具会更快成熟。你用的 Claude Code、Codex 等工具会受益于更好的训练和评估。
  • 开源社区的力量 — +13 票来自社区成员，说明这是真正的 grassroots 运动，不是大公司主导的。这意味着标准会更开放、更灵活。
🎯 今天就能做的事
  1. 读 HuggingFace 博客 — 去 huggingface.co/blog 看 "The Open Source Community is backing OpenEnv for Agentic RL" 文章。
  2. 了解 OpenEnv — 去 github.com/meta-ai/openenv 看项目文档，了解它支持哪些任务（代码、工具使用、多轮对话等）。
  3. 关注 agent 评测 — 如果未来有 agent 工具宣称在 OpenEnv 上达到某个分数，你可以用它来评估工具的好坏。
  4. 思考企业应用 — 如果你公司在评估 agent 平台，可以要求供应商提供 OpenEnv 上的评测结果。

跑在什么之上 — OpenEnv 是一个 Python 库，基于 Gymnasium（OpenAI Gym 的继承者）。它支持多种 agent 框架（LangChain、AutoGPT、OpenHands 等），并提供标准化的 API 和评估脚本。

上手难度 — 对开发者：pip install openenv 即可开始使用。对 clawii：不需要直接使用，但可以关注基于 OpenEnv 的评测结果。

诚实说缺点 — 标准化还处于早期阶段，任务覆盖不够全面。社区投票支持不等于正式标准。不同 agent 框架的适配程度不同。

你能拿它做什么 — 如果你在选择 agent 工具（Claude Code vs Codex vs OpenCode），可以看它们在 OpenEnv 上的评测结果。如果你在公司评估 agent 平台，可以要求供应商提供 OpenEnv 分数。
