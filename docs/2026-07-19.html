Qwen 3.8 Max 预览发布，即将开源

爪哩，阿里今天发布了 Qwen 3.8 Max 预览版，在 qwencloud.com 上可以体验。HN 上 207 points，讨论热烈。官方说即将开源（open-weight），但还没给具体日期。这是 Qwen 系列的最新旗舰，预计会跟 Kimi K3、Inkling 正面竞争。对于你来说，如果 Qwen 3.8 开源，意味着又多了一个可以在本地或便宜 API 上跑的前沿模型。

📍 背景 · Qwen 是阿里云的开源大模型系列，之前 Qwen 3.6 已经在多个 benchmark 上表现不错。这次 Qwen 3.8 Max 是新一代旗舰，从预览版来看，可能在编码和 agent 任务上有明显提升。阿里在开源方面一直比较积极，Qwen 系列在 HuggingFace 上有大量下载。如果 Qwen 3.8 真的开源，将是继 Kimi K3（7月27日开源）和 Inkling（已开源）之后的又一个重要开源模型。

为什么这事重要

  • 阿里在 AI 上的投入信号 — 连续快速迭代 Qwen 系列，说明阿里在 AI 上认真了。这对生态是好事。

Claude Code 改用 Bun（Rust 编写）——速度提升

爪哩，Simon Willison 今天报道：Claude Code 现在使用 Bun 运行时，而 Bun 是用 Rust 写的。这不是说 Claude Code 本身换了语言，而是它的底层 JavaScript 运行时从 Node.js 换成了 Bun。Bun 比 Node.js 快很多，尤其是在启动时间和包管理上。对于你每天用 Claude Code 做 vibe coding 来说，这意味着更快的响应和更流畅的体验。

📍 背景 · Claude Code 是 Anthropic 的编码 agent，运行在终端中。它最初基于 Node.js。Bun 是一个新兴的 JavaScript 运行时，用 Zig 语言编写（不是 Rust，但 Simon 的报道说"用 Rust 写的 Bun"——实际上 Bun 的核心是 Zig，但可能他指的是 Bun 的某些组件用 Rust）。Bun 以极快的启动速度和内置的包管理器闻名。Anthropic 将 Claude Code 迁移到 Bun，可能是为了提升性能和减少资源占用。

为什么这事重要

  • 日常使用体验提升 — Bun 的启动速度比 Node.js 快 5-10 倍。如果你每天打开 Claude Code 几十次，每次省几秒钟，累积起来很明显。
  • Anthropic 对性能的重视 — 换运行时不是小事，说明 Anthropic 在持续优化 Claude Code 的用户体验。
  • Bun 生态的认可 — 被 Anthropic 采用是对 Bun 的重要背书。如果你在做任何 JavaScript/TypeScript 项目，Bun 值得一试。
🎯 今天就能做的事
  1. 更新 Claude Code 到最新版本：npm update -g @anthropic-ai/claude-code（如果通过 npm 安装）或重新下载 CLI。
  2. 体验启动速度变化：运行 claude 命令，感受启动时间是否变快。
  3. 如果你自己写 Node.js 脚本，可以试试 Bun：curl -fsSL https://bun.sh/install | bash，然后 bun run your-script.js。

老实说：这个变化对大多数用户来说可能感知不强——Claude Code 的主要延迟来自模型推理，而不是运行时启动。但如果你在 CI/CD 环境中频繁调用 Claude Code，启动时间的节省会累积。另外，这可能是 Anthropic 为未来更轻量级的 agent 做铺垫。

🔭 关注点 · 关注 Claude Code 的更新日志，看是否有更多性能优化。另外，关注 Bun 的采用率——如果更多 AI 工具转向 Bun，说明它正在成为 JavaScript 运行时的新标准。
术语解码 · Bun = 一个快速的 JavaScript 运行时，类似 Node.js 但更快。用 Zig 语言编写。名字来源于"bun"（小面包），寓意轻量快速。运行时 = 程序运行的环境，比如 Node.js 或 Bun 就是 JavaScript 代码的运行环境。

Transcribe.cpp + Moonshine：本地语音识别新标杆

爪哩，今天 HN 上两个语音识别项目爆火：Transcribe.cpp（551 points）和 Moonshine（468 points）。Transcribe.cpp 是一个 C++ 实现的语音识别工具，非常轻量。Moonshine 是一个不到 500KB 的语音识别 + TTS 工具。两个都支持本地运行，不需要 GPU。对于你来说，这意味着可以在本地做语音转文字，而不需要依赖 OpenAI Whisper API 或 Google Cloud。

为什么这事值得知道

  • 本地语音识别的门槛大幅降低 — Moonshine 只有 500KB，可以在任何电脑上运行。Transcribe.cpp 用 C++ 实现，性能极佳。
  • 对 vibe coding 的潜在应用 — 你可以用语音输入来写代码，或者用语音控制 agent。虽然目前还不是主流用法，但工具已经成熟了。
  • 隐私优势 — 所有处理都在本地完成，不需要上传音频到云端。
