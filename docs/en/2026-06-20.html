GPT-5.5 hallucinates 3x more than MIT-licensed GLM-5.2 — independent comparison shows open model reliability edge

An independent comparison from arrowtsx.dev hit HN with 202 points and 61 comments. The core finding: GPT-5.5 hallucinates at a rate 3x higher than GLM-5.2, an MIT-licensed open model from Z.ai. This isn't to say GLM-5.2 is universally better, but on the crucial dimension of "doesn't make things up," the open model is ahead. A counterintuitive result worth your weekend reading.

📍 Context

GLM-5.2 is Z.ai's MIT-licensed open-weight model released June 16 — 753B parameters (40B active MoE), with a 1M token context window. It's been praised for coding and long-horizon tasks, but lacked an independent hallucination evaluation. This comparison fills that gap. GPT-5.5 is OpenAI's flagship, generally assumed to have among the best hallucination control of any model. That assumption may need updating.

Why this matters

  • Open model flips the reliability script — Conventional wisdom says closed models (GPT-5.5, Claude Opus) are more reliable on hallucination. This comparison shows GLM-5.2 significantly outperforming GPT-5.5. If replicated, it changes your mental model of which model to trust for factual work.
  • What this means for you — If you use AI for accuracy-sensitive tasks (data analysis, report generation, code review), hallucination rate is a critical metric. GLM-5.2 hallucinates less, meaning fewer "looks right but is wrong" outputs. That matters for any non-developer vibe-coding with AI.
  • GLM-5.2's ecosystem is already solid — Supported on Transformers, vLLM, SGLang, Cloudflare Workers AI, OpenRouter, Ollama Cloud, Baseten, DeepInfra, Fireworks, and more. You can use it via OpenRouter without self-hosting. Cost: ~$0.5/M tokens via OpenRouter vs GPT-5.5's $15/M — roughly 30x cheaper.

Methodology — The author used a standardized hallucination test suite covering factual Q&A, summary consistency, and code correctness. GLM-5.2 outperformed GPT-5.5 across nearly all dimensions, particularly on factual Q&A where its hallucination rate was ~3x lower.

Why GLM-5.2 might hallucinate less — Possible explanations: 1) GLM-5.2's training data may emphasize factual accuracy more; 2) Its architecture (MoE + IndexShare sparse attention) may naturally reduce hallucination; 3) GPT-5.5 may sacrifice some factuality for more "creative" outputs.

What this means for you — 1) For accuracy-sensitive text generation (reports, analysis, code review), GLM-5.2 may be the better choice; 2) If you need multimodal or complex reasoning, GPT-5.5 still has advantages; 3) GLM-5.2 costs ~1/30th of GPT-5.5 — remarkable value.

Honest caveats — 1) This is one independent test, not an official benchmark — needs replication; 2) GLM-5.2 is text-only — no image/multimodal support; 3) Its large context window (1M) may be less stable in practice than GPT-5.5; 4) GLM-5.2's ecosystem, while solid, isn't as mature as GPT-5.5's.

Hallucination rate
The frequency at which an AI model generates plausible-sounding but factually incorrect content. Lower is better for reliability.
MoE
Mixture of Experts — a model architecture where only a subset of parameters activate per token, improving efficiency.
IndexShare
A sparse attention mechanism pioneered by DeepSeek, used by GLM-5.2 to enable its 1M token context window.
