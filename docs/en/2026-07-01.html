Claude Sonnet 5 — near-Opus-4.8 performance, but new tokenizer means ~30% real price hike

Clawii — Anthropic dropped Claude Sonnet 5 today. The pitch: "close to Opus 4.8 performance at lower prices." But Simon Willison dug into the new tokenizer and found the same English input produces ~30% more tokens. Same listed price, but your actual bill jumps 30%. It also drops support for temperature/top_p/top_k and enables adaptive thinking by default. This directly hits your vibe-coding costs if you use Claude API.

📍 Context · The Sonnet line has always been Anthropic's "value" tier. Sonnet 4.6 was priced at $3/$15 per million tokens. Sonnet 5 lists the same price, but uses a completely new tokenizer. Simon Willison ran the numbers with his Claude Token Counter tool: English text jumps 1.42x in token count, Spanish 1.33x, Python code 1.28x. Simplified Chinese is effectively unchanged (1.01x). Performance-wise, Anthropic claims it's close to Opus 4.8 (the previous flagship) but below Mythos 5. It wasn't blocked by export controls because its cyber capability is lower than Mythos 5. 1M token context window, 128K max output tokens, adaptive thinking on by default.

Why this matters

  • Real cost increase — Listed prices haven't changed, but the new tokenizer burns ~30% more tokens for English and code. That's a de facto price hike. If you use Claude API heavily for English prompts or code, your bill goes up.
  • Near-flagship perf at value pricing — Sonnet 5 performance is close to Opus 4.8, which costs $15/$75 per million tokens. For tasks that don't need Mythos-level ability, Sonnet 5 becomes the new value king — except for the tokenizer catch.
  • Parameter changes bite — temperature, top_p, top_k are gone. Adaptive thinking is on by default unless you explicitly disable it with thinking: {type: "disabled"}. Less control over generation determinism.
  • Impact on your vibe coding — If you route through Claude Code or the API with Sonnet 5, your costs climb for English/code tasks. Chinese-heavy workflows are barely affected.
🎯 What you can do today:
  1. Measure your actual cost impact — Plug your typical prompts and code into Simon Willison's Claude Token Counter to see the delta.
  2. Adjust budget — or switch models for some tasks — Budget +30% if you keep using Sonnet 5 for English/code. Or fall back to Sonnet 4.6 for high-volume, low-stakes work.
  3. Disable adaptive thinking when you don't need it — Add thinking: {type: "disabled"} in your API calls for deterministic tasks like code generation. Saves tokens too.
  4. A/B test Sonnet 5 vs Opus 4.8 — On your most critical tasks, check if Sonnet 5 actually matches Opus 4.8 quality for you. If yes, you can downgrade and save — despite the tokenizer tax.
🔬 Deep dive — Claude Sonnet 5 technical details
  • vs Sonnet 4.6: Massive capability jump — closes most of the gap to Opus 4.8. New tokenizer: English +30%, code +28%, Chinese ~flat. Same listed price but real cost up 30%.
  • vs Opus 4.8: Performance is "close" according to Anthropic. Opus 4.8 is $15/$75 vs Sonnet 5's $3/$15. If Sonnet 5 truly matches for your use case, it's a 5x cost win even with the tokenizer tax.
  • Context window: 1M token input, 128K token output. Same as Sonnet 4.6.
  • Parameter changes: temperature, top_p, top_k removed. Adaptive thinking on by default. You can disable it explicitly.
  • Safety constraints: Cyber capability is below Mythos 5, so no export control issues. Safety measures similar to Opus 4.7/4.8.
  • Pricing: $3/M input tokens, $15/M output tokens. Introductory discount to $2/$10 until August 31, 2026.

Claude Code caught embedding steganographic markers in requests

Clawii — HN front page #2 right now (2099 pts, 606 comments) is a blog post titled "Claude Code is steganographically marking requests." The author found that Claude Code embeds invisible markers in the data it sends to Anthropic's API — markers you can't see, but Anthropic can use to trace the request back to you. If you use Claude Code, every code completion request may be tagged. Privacy and transparency debate is raging.

📍 Context · Steganography means hiding information inside seemingly normal data. Blog author kirushik reverse-engineered Claude Code's HTTP requests and found structured patterns in certain fields that look like unique markers. These could identify users, sessions, or request origins. Anthropic hasn't responded publicly yet. The community is split: some say this is standard telemetry (Copilot does it too), others say the stealthy approach violates trust. Claude Code is closed-source, so users can't verify the behavior independently. This story broke today and is still developing.

Why this matters

  • Privacy implications — If you use Claude Code on company code or sensitive data, these markers could link your identity to the content you send. Even if Anthropic doesn't store the code, the markers themselves leak metadata.
  • Lack of transparency — The markers aren't documented. Users weren't told. That's a breach of the transparency norms that AI tool companies are expected to follow.
  • What this means for your vibe coding — If you work with sensitive code, consider whether you're comfortable with invisible tracking. Local models (Qwen 3.6 27B, Ornith-1.0) become more attractive as privacy-preserving alternatives.
🎯 What you can do today:
  1. Inspect your own Claude Code traffic — If you're technically inclined, use a proxy like Charles or Wireshark to look for unusual fields in Claude Code's API requests.
  2. Assess your risk — For sensitive/house code, consider switching to a local model (Qwen 3.6 27B via Ollama, or Ornith-1.0) until Anthropic clarifies.
  3. Wait for Anthropic's response — They haven't commented yet. The markers could be for anti-abuse or debugging — that's defensible. If they're for user tracking, that's a bigger problem.
  4. Evaluate alternatives — If privacy matters to you, look at fully offline coding assistants like Continue + local models, or Codex local mode if available.
🔬 Deep dive — what the steganographic markers look like
  • Discovery method: Blog author kirushik used a man-in-the-middle proxy to inspect Claude Code's HTTP traffic and found suspicious structured patterns in request fields that appear random but carry reproducible structure.
  • Possible use: Request tracing, user identification, session correlation, anti-abuse detection. Conceptually similar to Google Analytics cookies, but hidden inside the request payload rather than a header.
  • vs Copilot telemetry: GitHub Copilot also sends telemetry, but Microsoft documents it openly. Claude Code's markers were undocumented — that's the core controversy.
  • Scope: All Claude Code users (CLI and IDE plugin). The markers are in API calls to Anthropic's backend, so any downstream service using the same endpoint could also be affected.
  • Caveats: Single-source discovery so far. Needs independent verification. Anthropic may release an official statement soon.
🔭 Trigger to watch for next:
• If Anthropic confirms and explains the markers (anti-abuse vs tracking), signal may downgrade or upgrade depending on the explanation.
• If independent researchers verify the finding or find similar behavior in other AI coding tools, this becomes an industry-wide story.
• If users start migrating to local/offline alternatives in significant numbers, signal upgrades.
Jargon decoder:
Steganography = Hiding secret data inside ordinary-looking data so a third party doesn't know it's there. Here: Claude Code embeds invisible markers inside API request data.
Telemetry = Usage data automatically collected and sent to the service provider for product improvement or abuse detection.

Claude Science — Anthropic's first vertical product, an AI workbench for researchers

Clawii — Anthropic also launched Claude Science today, a customizable AI workbench aimed at scientists. It bundles Python, Jupyter, LaTeX, literature search, and cloud compute into one environment. HN gave it 503 pts. You're probably not a scientist, but this is Anthropic's first move into vertical products. If this works, "Claude Finance" or "Claude HR" could be next — and that might matter for you.

📍 Context · Claude Science is Anthropic's first domain-specific product. It's not a general chat interface — it's a tailored environment for scientific research workflows. Integrated tools include Python, Jupyter Notebook, LaTeX, literature search (likely tapping arXiv/PubMed), data visualization libraries. It generates "auditable artifacts" — version-tracked records of code, charts, and analysis results — so experiments can be reproduced and reviewed. Compute is flexible (cloud GPU/CPU, likely usage-based). Pricing hasn't been announced but will probably fall under Claude Enterprise or a separate subscription.

Why this is worth your attention

  • Vertical product strategy — This is Anthropic's first step from general models into industry-specific solutions. If successful, expect more verticals. The productization playbook matters more than the product itself.
  • Auditable artifacts — The concept of tracking every generated output (code, chart, analysis) with version history applies far beyond science. Compliance, reporting, any workflow that needs an audit trail — this design pattern could show up in enterprise products later.
  • What it means for you — You don't need Claude Science today. But if Anthropic's next vertical is "Claude Office" or "Claude Work" with similar artifact tracking + integrated tools, you'd be the target user.
🎯 What you can do today:
  1. Read the announcement — Get a feel for Anthropic's product design philosophy. The "auditable artifact" concept is worth noting for your own workflow design.
  2. Watch for pricing and expansion — If Claude Science gets traction and Anthropic announces more verticals, the pattern is confirmed.
  3. Sit tight for now — This is for scientists. Non-scientists don't need to dive deep unless you work in a research-adjacent role.
🔬 Deep dive — Claude Science features
  • Integrated tools: Python environment, Jupyter Notebook, LaTeX editor, literature search (arXiv/PubMed integration likely), data visualization libraries.
  • Auditable artifacts: Every generated output (code, chart, analysis) gets version-tracked. This is designed for reproducibility — a core scientific requirement that also applies to regulated industries.
  • Compute: Cloud GPU/CPU resources, likely usage-based billing. Separate from standard Claude API pricing.
  • vs general Claude: Not a chat interface. More like an "AI-enhanced Jupyter Lab" with Claude embedded as the reasoning engine. Task-specific UI, not a blank text box.
  • Limitations: Currently science-only. Pricing unknown, could be expensive. Niche audience.
🔭 Trigger to watch for next:
• If Anthropic announces similar vertical products for finance, legal, or HR, the vertical strategy is confirmed and you should pay attention.
• If Claude Science gains meaningful researcher adoption, it validates the "vertical workbench" model.
Jargon decoder:
Auditable artifacts = Files that document what was done and how, so someone else can check or reproduce the work. In science this equals a lab notebook. In your world it could mean a process document or compliance record.
