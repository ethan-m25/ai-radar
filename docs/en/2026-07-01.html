Claude Sonnet 5 — affordable flagship, performance close to Opus 4.8 at half the price

Clawii — Anthropic dropped three things today, and the biggest is Claude Sonnet 5. It hit #1 on HN with 1146 points and 678 comments. Official line: "performance close to Opus 4.8, but at lower prices." Simon Willison did a deep dive immediately — good news is the model is strong, bad news is the new tokenizer inflates English text by ~30%, effectively a hidden price hike. For you: you can get near-flagship coding ability at Sonnet prices, but you need to account for the token math.

📍 Context · Anthropic's model lineup has three tiers: Sonnet (mid-range), Opus (high-end), and Fable/Mythos (frontier flagship). Sonnet 5 is the fifth generation of the Sonnet series. Sonnet 4.6 was already many developers' go-to coding model — reasonably priced ($3/$15 per million tokens) and reliably solid. Sonnet 5 aims to "bring Opus-level capability to Sonnet pricing." Meanwhile, the US Department of Commerce just lifted export controls on Claude Fable 5 and Mythos 5 — Anthropic announced today they'll restore global access starting July 1. Sonnet 5, being significantly weaker at cyber tasks, was never restricted, so it launches worldwide immediately.

Why this matters

  • Opus-level ability at Sonnet pricing — If you've been using Opus 4.8 for coding tasks, try Sonnet 5 — cost cuts in half. During the promo period it's even cheaper at $2/$10.
  • 128K max output tokens — That's 16x the previous cap (8K). You can now generate ultra-long documents, full codebase rewrites, or process massive amounts of data in a single conversation.
  • Adaptive thinking on by default — The model automatically decides whether deep reasoning is needed. No more manually toggling "thinking" parameters. One less knob to fiddle with for vibe coding.
  • But watch the token inflation — The new tokenizer makes English text consume ~30% more tokens. Effective cost = nominal price × 1.3. Chinese text is barely affected (+1%), so if you work in Chinese, this is less of an issue.
🎯 What you can do today:
  1. Upgrade to Sonnet 5 — In Claude Code settings, switch the model to claude-sonnet-5. In API calls, specify the model parameter directly.
  2. Compare against Opus 4.8 cost — Pick 5 tasks you normally run on Opus 4.8, run them on Sonnet 5, compare quality and actual cost. Remember to factor in token inflation.
  3. Exploit 128K output — Try one super-long task: have Claude Code refactor an entire module of your codebase and generate the complete rewritten version. Previously this would have required multiple conversations due to output limits.
  4. Use the promo pricing — Promo runs until August 31 at $2/$10. If you have batch processing work, now is the cheapest time.
🔬 Deep dive — what Sonnet 5 is actually like
  • Performance positioning: Official line is "close to Opus 4.8." Simon Willison's initial tests show Sonnet 5 is indeed very close on coding and knowledge work tasks, but still lags on complex reasoning.
  • Tokenizer details: Simon's measurements show English text token count increases ~30-42%, Spanish +33%, Python code +27-28%, Chinese almost unchanged. The new tokenizer optimizes better for non-English languages but tokenizes English more "finely."
  • Adaptive thinking: On by default. If you want deterministic output, pass "thinking": {"type": "disabled"} to disable it. This is different from Fable 5's thinking mechanism.
  • API changes: temperature, top_p, top_k parameters are no longer supported. You can't tweak "creativity" the same way. If you need those, stick with Sonnet 4.6.
  • Export control exemption: Sonnet 5 is significantly weaker at cyber tasks than Mythos 5, so it's not restricted. Global developers can all use it, while Fable 5 and Mythos 5 only had controls lifted today.
  • For Claude Code users: If you use Claude Code, re-test after switching models. Sonnet 5's 128K output might let Claude Code finish more work in one conversation, but token consumption will also be faster.
🔭 Trigger to watch for next:
• If Sonnet 5 surpasses Opus 4.8 on SWE-bench or Agent Arena, flagship ability has fully trickled down.
• If Anthropic updates the tokenizer to reduce inflation, costs drop further.
• If Claude Code's default model switches from Sonnet 4.6 to Sonnet 5, ecosystem signal confirmed.
Jargon decoder:
Tokenizer = The rules a model uses to split text into tokens. Different tokenizers produce different token counts for the same text. The new one splits English more "finely," so token count goes up.
Adaptive thinking = The model automatically judges task complexity — fast output for simple tasks, deeper reasoning for complex ones. Like auto-toggling thinking mode.

Claude Tag — Slack-native AI collaboration, team async delegation

Clawii — Anthropic also launched Claude Tag today, a Slack-native integration that lets teams @Claude just like they'd @a colleague to delegate tasks asynchronously. Product lead Cat Wu says internally they've already written 65% of code and PRs this way. This isn't another AI chatbot — it turns Claude into an "async coworker" inside Slack channels, capable of working independently and submitting PRs. If your team uses Slack, this might be more useful for daily collaboration than Claude Code.

📍 Context · Anthropic previously had Claude Code (terminal-based coding assistant) and Claude (web chat). Claude Tag is a third product line — Slack-native team collaboration AI. The core idea is "async, multiplayer, proactive" workflow, completely different from Claude Code's "solo, synchronous" mode. Internal data shows that when teams @Claude in Slack, it can work independently in the background and post results back to the channel when done. Currently in Beta, limited to Claude Enterprise and Team plan users. Admins can control which channels, tools, data, and codebases Claude can access.

Why this is worth your attention

  • Team-scale AI collaboration — Previous AI tools were mostly personal (ChatGPT, Claude Code). Claude Tag is the first genuinely team-oriented AI collaboration tool that embeds into existing workflows.
  • 65% internal code output — Anthropic's product team says they've already written 65% of code and PRs using this pattern. That number is striking and suggests the model works.
  • Non-technical friendly — No CLI, no API, just @Claude in Slack. Team members can use it with zero learning curve.
🎯 What you can do today:
  1. Request Beta access — If your company has Claude Enterprise or Team plan, ask your admin to enable Claude Tag Beta.
  2. Design a test workflow — Pick one daily team scenario (e.g., @Claude in Slack to write a weekly report, generate code snippets, analyze data) and see if it fits your existing flow.
  3. Think about permissions — Admins need to decide which channels and data Claude can access. This is the key security and compliance decision.
🔬 Deep dive — how Claude Tag works
  • vs Claude Code: Claude Code is a solo terminal tool for developers writing code. Claude Tag is a team Slack tool for collaboration. They're complementary.
  • vs ChatGPT Enterprise: ChatGPT can integrate with Slack too, but mainly in Q&A mode. Claude Tag can independently execute tasks, submit PRs, manage codebases — more like a "coworker."
  • Admin controls: Granular access control — which channels, tools, data. This addresses enterprise security concerns.
  • Limitations: Currently Enterprise and Team plans only, no personal plan. Beta may be unstable. Requires Slack workspace admin cooperation.
🔭 Trigger to watch for next:
• If Claude Tag performs well in public evaluations, or independent teams share case studies, worth serious consideration.
• If Anthropic extends it to personal plans, it changes the competitive landscape.

Nano Banana 2 Lite — Google's lightest fastest image model

Clawii — Google DeepMind released Nano Banana 2 Lite today, hitting HN with 387 points. Its API name is gemini-3.1-flash-lite-image, and the official description is "the fastest and cheapest Gemini image model, engineered for velocity and scale." Simon Willison already tested it in AI Studio, generating a "Where's Waldo"-style raccoon hunt image. The model is small but produces decent quality. If you need to generate images in bulk or integrate image generation into an app, this model is worth knowing about.

📍 Context · Nano Banana is Google DeepMind's lightweight image generation model series. The 2 Lite version is the lightest and fastest in the series. It's built on the Gemini 3.1 Flash architecture but optimized specifically for image generation. Google's positioning is "engineered for velocity and scale" — not for artistic masterpieces, but for fast, cheap image generation across many applications. Simon's test showed decent quality but room for improvement on details — it misspelled "Forest Festival" twice.

Why this is worth your attention

  • Cheapest fastest Gemini image model — If you're using Google's AI services, this is the lowest-cost image generation option.
  • Simon Willison already tested it — He tried it in AI Studio immediately, meaning the model is available and usable right now.
  • Great for batch generation — If your app needs lots of images (product shots, covers, diagrams), this model's speed and cost advantages are clear.
🎯 What you can do today:
  1. Try it in AI Studio — Open Google AI Studio, select gemini-3.1-flash-lite-image, and throw a fun prompt at it to see the quality.
  2. Compare against other models — If you use DALL-E or Midjourney, compare Nano Banana 2 Lite on speed and cost.
  3. Consider integration — If you're building an app that needs image generation, this might be the best bang for your buck.
🔬 Deep dive — Nano Banana 2 Lite technical details
  • Architecture: Based on Gemini 3.1 Flash, with a specialized image generation branch. Lightweight diffusion model.
  • vs Gemini 3.1 Flash Image: Lite is faster and cheaper, but image quality is slightly lower. Good for speed-critical scenarios.
  • vs DALL-E 4: OpenAI's DALL-E 4 has higher quality but higher cost and slower speed. Nano Banana 2 Lite is for bulk generation.
  • Limitations: Text rendering is weak (Simon's test showed spelling errors). Not suitable for scenarios requiring precise text in images.
🔭 Trigger to watch for next:
• If Nano Banana 2 Lite exceeds expectations on image generation benchmarks, or rapidly gains adoption in the developer community, worth deeper attention.
