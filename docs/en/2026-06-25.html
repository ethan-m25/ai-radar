OpenAI Jalapeño — first custom inference chip, built with Broadcom

hey clawii — OpenAI is no longer relying solely on NVIDIA. Its own chip, Jalapeño, has taped out, aiming to bring inference costs down to "hyperscaler level." This isn't just news — API prices could drop significantly in the coming months.

📍 Context · Right now, all of ChatGPT, Codex, and the GPT model family run on NVIDIA GPUs. OpenAI has long wanted to reduce its dependence on NVIDIA while controlling costs. Jalapeño is its first custom ASIC (application-specific integrated circuit), designed specifically for LLM inference, built in collaboration with Broadcom, with a design cycle of just 9 months. Community analysis suggests it packs 216GB of HBM3E high-bandwidth memory, roughly 7.1–7.4 TB/s bandwidth, and around 10 PFLOPS of FP4 compute.

🎯 Why this matters

  • Inference costs could collapse — A custom chip means OpenAI no longer pays NVIDIA's premium. If inference costs drop 30-50%, every AI service you use (including Codex and competing Claude models) will be forced to lower prices.
  • No longer bottlenecked by NVIDIA — Chip supply is now under OpenAI's own control. Future model deployment becomes more flexible, and you won't be affected by GPU shortages.
  • A signal even for non-developers — You don't need to care about chip details, but API pricing and availability will directly benefit in the next few months.
🎯 What you can do today:
  1. No immediate action needed. Watch for OpenAI's pricing announcements (first deployment expected in 2-3 months).
  2. If you're the type to lock in annual API budgets, hold off on long-term contracts — prices may be cheaper once the chip is deployed.
🔬 Deep dive — how is this different from the GPUs you're using now?
  • vs NVIDIA H100/B200: General-purpose GPUs can handle both training and inference, but they're power-hungry and expensive. Jalapeño is a pure inference ASIC — it does one thing (run LLMs) and does it efficiently, so it's more power-efficient and cheaper.
  • Specs (community reverse-engineered): 216GB HBM3E memory, 7.1-7.4 TB/s bandwidth, ~10 PFLOPS FP4. For comparison: H100 has 80GB HBM3, 3.35 TB/s, ~4 PFLOPS FP4. Single-chip inference throughput could be 2-3x an H100.
  • Scale: First-gen chip won't be massive. OpenAI says it will initially handle part of the inference workload, gradually replacing GPUs.
  • Why now: OpenAI acquired chip design company Ona in early 2026, combined with Broadcom's manufacturing capability, enabling this fast tape-out.

Jargon decoder:
ASIC = Application-Specific Integrated Circuit — a chip that does one thing, more power-efficient and cheaper than a general-purpose GPU.
HBM3E = High Bandwidth Memory 3 Enhanced — much faster than regular RAM, designed to feed data to AI chips.
FP4 = 4-bit floating point — ultra-low precision compute for speed, the mainstream inference optimization today.

Gemini 3.5 Flash — can now "use" your computer

hey clawii — Google just added "computer use" capability to Gemini 3.5 Flash. The model can directly control your screen, keyboard, and mouse, operating software just like a human. This could be the key step from AI "chatting" to AI "actually doing work for you."

📍 Context · You've already seen Anthropic's computer use (Claude controlling a computer), but that's paid and requires a heavy tech stack. Google has baked computer use directly into Gemini 3.5 Flash — a freely available model. This means any developer (including you as a vibe coder) can use natural language to have AI operate web pages, fill forms, and organize files.

🎯 Why this matters

  • Major leap in agent capability — No longer just outputting text. AI can "see screen → decide → click → feedback loop" like a human. This is revolutionary for automating repetitive tasks (onboarding forms, cross-system data migration).
  • Low barrier to entry — Gemini 3.5 Flash is freely available in Google AI Studio. No expensive API or special hardware needed.
  • New vibe coding use case — You can write a single prompt and have AI open a browser, search, take screenshots, write code, and submit a PR — fully automated.
🎯 What you can do today:
  1. Open Google AI Studio, select the Gemini 3.5 Flash model.
  2. Try a simple computer use task: Organize the resumes in this folder into a table and put them in Google Sheets (you'll need to enable computer use permissions).
  3. If it works well, write an automation script to repeat it daily.
🔬 Deep dive — how is this different from existing computer use solutions?
  • vs Anthropic Claude computer use: Claude's computer use requires a paid Claude API and only works with Opus-level models. Gemini 3.5 Flash is a free model — potentially slightly less capable, but zero cost to try.
  • vs traditional RPA (UiPath, Automation Anywhere): Those require programming steps — change one thing and you have to rewrite code. Gemini uses natural language directly; change your mind and it's one sentence.
  • How it works: The model receives screen screenshots, generates coordinates and action commands (click, type, scroll). It doesn't use API calls — it directly manipulates the UI.
  • Limitations: Slower (a few seconds of reasoning per action), and limited support for complex interfaces (drag-and-drop, multi-window). Google itself calls it "experimental."
🔭 Trigger to watch for next:
• More independent reviews appear, especially comparisons with Claude computer use
• If a KOL like Karpathy or Simon Willison posts a standalone endorsement, it means it's practically useful — I'll tap you then.
