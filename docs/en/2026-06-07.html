MetaGPT v0.8 makes agent teams work like a meeting, not a script

Open-source multi-agent framework MetaGPT released v0.8 today with natural language orchestration, parallel sub-agents, and a dynamic workflow engine. You don't need to write code to define an entire "agent team" that works for you.

📍 Context

MetaGPT is the 2nd most-starred AI agent framework on GitHub (behind LangChain). Its big idea: simulate a software company. You define roles — product manager, architect, engineer — and they collaborate to build things. Before v0.8, orchestrating those agents meant writing Python or YAML config files. The v0.8 change? You describe what you want in natural language, and MetaGPT figures out how to split the work, who does what, and in what order.

Why this matters

  • Zero-code orchestration — No scripting required. For clawii — someone who "vibe-codes" with Claude Code and Codex — this means you can tell MetaGPT "write me a competitive analysis report, then make a slide deck, then check the data for consistency" and it auto-splits the work across agent roles.
  • Parallel sub-agents — Older versions ran agents one-at-a-time: one agent finishes, next starts. Now multiple agents can work simultaneously — one searches, one writes code, one makes charts — then merge results. For complex workflows, speed roughly doubles.
  • Dynamic workflows — Agents can "change the plan" mid-stream based on intermediate results. If writing code reveals you need to look up an API doc first, the agent can insert a search task on the fly and keep going.
🎯 Action — what you can do today
  1. Install — pip install metagpt==0.8.0 (requires Python 3.11+ and an OpenAI or Claude API key).
  2. Try the simplest team — Run metagpt --run "build a web-based countdown timer in one HTML page with CSS and JS". It auto-spawns PM, architect, and engineer roles.
  3. Customize a role — Add role: check_quality in config.yaml and say "I am a test engineer who checks edge cases." The agent auto-runs a quality check on every output.
  4. Read the official demo — Check the examples/nl_orchestration_guide.md in the GitHub repo. It has 5 examples from simple to complex using natural language.

What it runs on — MetaGPT works with any OpenAI-compatible API (OpenAI GPT-4o/GPT-5.5, Anthropic Claude, OpenRouter models). v0.8 defaults to GPT-4o (API key required), but you can switch to Claude or local models via environment variables LLM_API_KEY and LLM_API_BASE. Stack: pure Python 3.11+, depends on pydantic, httpx, jinja2.

How hard to use — Install: 2 minutes (pip). Configure API key: 30 seconds. First run of a simple task: 5-10 minutes (MetaGPT starts multiple roles and waits for their conversation). Complex tasks need tuning (role count, model choice, max tokens), but v0.8's natural language mode keeps most scenarios code-free.

Honest caveats — Token consumption is high (multi-agent chat rounds add up). Speed depends on API latency (parallel agents are faster but double API calls). Long document generation can get stuck in loops. It's not built for real-time interaction — better for background task processing.

What you can actually do with it — One command: an "analyst agent" researches + a "writer agent" drafts a report + a "quality agent" checks facts. Or: an "architect agent" designs the system + an "engineer agent" writes code + a "test agent" runs tests + a "deploy agent" pushes to a server.

Orchestration
Organizing multiple AI agents to work together — like a director with actors. You don't talk to one AI; you define a cast of AI roles and they auto-collaborate.
Harness
In agent engineering, the "layer on top of the model" — prompt templates, tool calling, error retries, workflow control. MetaGPT is a harness framework, not a model.
