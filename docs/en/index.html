Claude Tag — Slack-native team collaboration, async delegation to Claude

hey clawii — Anthropic stuffed a full Claude into Slack. You can @-mention it in any channel and assign tasks. This isn't a simple chatbot — it's a "multiplayer, async, proactive" workflow layer.

📍 Context · You already use Claude Code as your personal coding agent, but team collaboration has been a blind spot — everyone has their own conversation, info doesn't sync, tasks don't align. Claude Tag runs directly inside Slack, so the team can assign work to Claude just like they'd assign it to a person. It can collaborate across channels and proactively remind you. Internally, the product team used it to write 65% of their code and PRs.

🎯 Why this matters

  • Team-level AI workflow — No longer just personal AI use. The whole team shares one AI colleague. Admins control which channels, tools, and codebases Claude can access.
  • Async + proactive — Claude processes tasks in the background and @-mentions you when done. No need to stay in the channel waiting.
  • Useful for non-developers too — You can drop Claude into your HR channel, finance channel — have it analyze data, write reports, review documents.
🎯 What you can do today (if you have a Claude Enterprise/Team plan):
  1. Log into your Slack workspace, install the Claude Tag app from the App Directory.
  2. In any channel, type @Claude review the terms in this contract (if your admin has granted access).
  3. Create a dedicated #ai-workflow channel and assign recurring tasks (weekly report summaries, data整理) to Claude permanently.
🔬 Deep dive — how is this different from existing Slack AI bots?
  • vs Slack AI (built-in): Slack AI can only do conversation summaries and search. Claude Tag can execute complex tasks — write code, analyze data, generate files — and proactively return results.
  • vs custom bots (Zapier/Workato): Those require you to pre-configure workflows. Claude Tag uses natural language in the channel to "assign tasks" — it's like sending a Slack message to Claude.
  • Core capability: Read/write Slack messages, access authorized files and codebases, run async background tasks. Claude can participate in multiple channels simultaneously without mixing contexts.
  • Currently only available through Claude Enterprise and Team plans, beta stage.

VibeThinker-3B — Weibo AI's small model punches up, new local vibe coding option

hey clawii — A 3B-parameter open-source model hit 41k HuggingFace downloads in 5 days with a surge in stars. It could become your new go-to local model for vibe coding.

📍 Context · You're already running Qwen3.6-27B locally for coding tasks, but 27B parameters demand serious GPU power (at least 16GB VRAM). VibeThinker-3B is just 3B parameters, designed to run smoothly on 8GB VRAM or even pure CPU, while maintaining solid coding and agent capability. From the Weibo AI team (WeiboAI) — not a big lab, but the quality seems legit: it shot to #5 on the HF trending leaderboard.

🎯 Why this matters

  • Local deployment barrier drops dramatically — 3B parameters run on consumer GPUs (6-8GB) or even M-series Macs via MLX. You no longer need to fight for GPU time just to run a coding assistant.
  • Optimized for vibe coding — The name itself hints at its target use case. From the docs, it excels at code completion, bug fixing, and small script generation.
  • Open source (MIT) — You can modify it freely and integrate it into your own workflow.
🎯 What you can do today:
  1. Open Ollama (if you haven't installed it: curl -fsSL https://ollama.com/install.sh | sh)
  2. Pull the model: ollama pull WeiboAI/VibeThinker-3B
  3. Point your vibe coding script to this model (e.g., switch backends in Claude Code or Cursor) and test daily tasks: fix a Python snippet, write a script, tweak some CSS.
  4. Compare response speed and output quality against Qwen3.6-27B. Keep a log.
🔬 Deep dive — how is this different from existing small models?
  • vs Qwen3.6-27B: Qwen is strong at coding but needs 16GB+ VRAM. It runs on a full M2 Ultra machine but not on portable devices. VibeThinker is 1/9th the parameters — runs on 8GB GPUs or even 8GB unified memory on Apple Silicon, and generates faster.
  • vs Gemma 4 12B: Gemma 4 12B is also a small model, but VibeThinker is smaller (3B) and specifically optimized for coding vibe. Gemma 4 leans more toward general conversation.
  • vs Llama 3.2 3B: Llama 3.2 3B is Meta's base model, but VibeThinker's download/star growth on HF is much faster, suggesting the community finds it more practically useful.
  • Technical details: Based on LLaMA architecture, 3B parameters, supports up to 32K context. Training data includes lots of Python, JavaScript, and shell scripts. MIT license.
