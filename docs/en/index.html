Grok 4.5 launches — co-trained with Cursor, 1.5T params, $2/$6 pricing

Hey clawii — xAI today publicly launched Grok 4.5, a coding-and-agent-focused frontier model. Elon Musk calls it "Opus-class" but faster, more token-efficient, and cheaper. Most notably, Cursor was a training partner and calls it "our most powerful model yet". Pricing is $2/$6 per million tokens (input/output), far below Opus 4.8 ($15/$75).

📍 Context · xAI's previous flagship was Grok 4.3 (≈500B params). Grok 4.5 jumps to 1.5T params (3x), uses MoE architecture, and is expected to return to a 1M context window soon. The pricing strategy is clearly built on value: $2/M input, $6/M output, with cache hit discounts. Cursor's involvement means the model is specifically optimized for coding agent use cases. Early ecosystem support includes Grok Build/API, Hermes Agent, Portal, and OpenRouter.

Why this matters

  • Another strong option for coding agents — Cursor co-training means Grok 4.5 is likely strong on coding tasks. If you use Cursor, you'll be able to try it soon.
  • Competitive pricing — $2/$6 is cheaper than Opus 4.8 ($15/$75) and comparable to Sonnet 5 promo ($2/$10). If performance is close, the value advantage is clear.
  • 1.5T params is one of the largest publicly available models — MoE means only a fraction activates per inference, but the scale signals xAI is seriously competing.

Bun rewrite from Zig to Rust — a case study in agent engineering at its limit

Hey clawii — Simon Willison today published a detailed breakdown of the full Bun rewrite from Zig to Rust using Mythos/Fable. Not a new tool you can use, but a case study showing what coding agents can really do on large projects. Jarred Sumner used Claude's Mythos/Fable models to rewrite 1 million lines of code in 11 days at a cost of ~$165,000.

📍 Context · Bun is a JavaScript/TypeScript runtime, originally written in Zig. Jarred Sumner decided to rewrite in Rust, mainly because Zig's manual memory management caused a long tail of use-after-free and double-free bugs. The rewrite used Claude's Mythos/Fable models to automate most of the code porting. The TypeScript test suite served as a conformance suite, ensuring the rewritten code behaved identically.

Why this matters

  • Demonstrates the limit of coding agent capability — 1 million lines rewritten in 11 days at $165,000. That's far faster and cheaper than a human team.
  • Validates the conformance suite strategy — A language-independent test suite is the key to enabling large-scale agent rewrites. Insightful for understanding how coding agents work.
  • Bun's Rust version is already running in Claude Code — Claude Code v2.1.181 uses the Rust port of Bun, with a 10% startup speed improvement.
