Apple Siri AI runs on Google Gemini + Google Cloud + NVIDIA GPUs

Apple announced at WWDC 2026 that Siri AI is powered by Google's Gemini models, running on Google Cloud infrastructure with NVIDIA GPUs. 611 points on HN. This marks Apple's first public admission that its in-house LLM capability wasn't enough, and a huge endorsement of Gemini as a platform.

📍 Context

Apple announced Apple Intelligence at WWDC 2024 with big promises, but delivery fell short. This year, Apple admitted their own small models weren't sufficient for demanding tasks. Instead, they're licensing a custom Gemini-derived model that runs on Google Cloud + NVIDIA hardware, protected by Apple's Private Cloud Compute (PCC) architecture. The security blog confirms PCC is now extended to third-party cloud infrastructure for the first time.

Why this matters

  • Apple concedes on in-house AI — For the first time, Apple publicly acknowledges that third-party models beat their own. This is a huge win for Google and a signal to OpenAI and Anthropic about competitive pressure.
  • Gemini gets the ultimate endorsement — If Apple trusts Gemini for Siri AI, the model is enterprise-grade. For clawii evaluating models, Gemini just got the strongest possible validation.
  • Privacy architecture breakthrough — PCC extended to Google Cloud while maintaining Apple's security promise. This sets a precedent for enterprise AI deployments on third-party clouds with strong privacy guarantees.
🎯 Action — what you can do today
  1. Read Apple's security blog — Go to security.apple.com for "Expanding Private Cloud Compute" — details on how they maintain privacy on Google Cloud.
  2. Try Gemini models — In Google AI Studio or via API, test Gemini 3.5 or Gemini Omni against GPT-5.5 and Claude Opus 4.8. See how they compare for your vibe coding tasks.
  3. Watch for Siri AI reviews — Once iOS 27 developer beta reviews land, pay attention to how well Siri AI actually performs. If it's good, Gemini excels at agentic tasks.
  4. Think about enterprise deployment — If you're involved in AI procurement at work, Apple's PCC-on-Google-Cloud model is a reference architecture for safe third-party AI deployment.

What it runs on — Siri AI uses a custom Gemini variant, inferred on Google Cloud's NVIDIA GPU clusters. Apple's Core AI Framework provides PyTorch integration — the coreai-torch package lets you deploy PyTorch models on Apple hardware by mapping ATen operators to Core AI operations.

How hard to use — Regular users: wait for iOS 27 update. Developers: Core AI Framework requires Xcode + iOS 27 SDK (medium learning curve). The PyTorch integration is straightforward if you already use PyTorch.

Honest caveats — Apple hasn't fully abandoned in-house AI, just conceded on general-purpose LLMs. Siri AI currently supports English only; Chinese timeline unannounced. Extending PCC to Google Cloud introduces supply chain risk.

What you can actually do with it — If you use an iPhone, Siri AI will handle complex tasks: booking restaurants, managing schedules, analyzing documents. If you're an AI developer, Core AI Framework lets you run models efficiently on Apple hardware.

Xiaomi MiMo-v2.5-Pro-UltraSpeed — 1T model, 1000 tok/s

Xiaomi released MiMo-v2.5-Pro-UltraSpeed, a 1T-parameter MoE model hitting 1000 tokens per second inference speed. 580 points on HN, 423 comments. This is Chinese vendors pushing inference speed to a new level — potentially upending API pricing globally.

📍 Context

Xiaomi's MiMo family is their MoE model line. The v2.5-Pro-UltraSpeed is a speed-optimized variant — likely using aggressive MoE routing + quantization + speculative decoding to hit 1000 tok/s. 1T total parameters but only a fraction active per inference (est. 50-100B). Speed is becoming the new moat as model precision converges.

Why this matters

  • Speed is the new moat — As model accuracy plateaus, speed becomes the key differentiator. 1000 tok/s enables real-time conversation, instant code completion, and high-throughput batch processing.
  • Direct impact on vibe coding — For clawii using APIs daily, speed directly affects workflow. 1000 tok/s is 5-10x faster than GPT-5.5 Pro, likely at a fraction of the cost.
  • Chinese vendor momentum — Xiaomi, DeepSeek, Alibaba are all pushing inference optimization hard. The global API pricing landscape could shift significantly within 6 months.

👀 Worth a Look · Agentic RL Standard Environment

OpenEnv — open-source community backs it as standard for agentic RL

The HuggingFace blog announced OpenEnv received +13 community votes as the standard environment for agentic reinforcement learning. This matters for anyone building or evaluating AI agents — standardized testing means faster progress and fair comparisons.

📍 Context

Agentic RL trains AI agents through trial-and-error to complete tasks. OpenEnv, open-sourced by Meta, provides standardized tasks and metrics — like OpenAI Gym but specifically for agent tasks (code generation, API calls, multi-turn dialogue, web browsing). The community vote to make it the standard is a grassroots movement, not a corporate dictate, meaning the standard will be more open and flexible.

Why this matters

  • Standardization signals ecosystem maturity — When agent development has a standard test environment, teams can compare fairly and progress accelerates. Same role as ImageNet did for computer vision.
  • Indirect impact on clawii — You don't do RL yourself, but standardized environments mean agent tools (Claude Code, Codex) will improve faster through better training and evaluation.
  • True open-source community momentum — +13 votes from community members, not a single company. This grassroots legitimacy makes the standard more durable and flexible than a vendor-led initiative.
🎯 Action — what you can do today
  1. Read the HuggingFace blog — Go to huggingface.co/blog for "The Open Source Community is backing OpenEnv for Agentic RL."
  2. Browse the OpenEnv repo — Go to github.com/meta-ai/openenv to see which tasks are supported (code, tool use, multi-turn dialogue, etc.).
  3. Track agent benchmarks — If future agent tools claim scores on OpenEnv, you can use that to evaluate tool quality.
  4. Consider enterprise evaluation — If your company is evaluating agent platforms, ask vendors for OpenEnv benchmark results.

What it runs on — Python library built on Gymnasium (OpenAI Gym successor). Supports multiple agent frameworks (LangChain, AutoGPT, OpenHands). Standardized API and evaluation scripts.

How hard to use — For developers: pip install openenv. For clawii: no direct use needed — just track OpenEnv-based benchmarks when comparing agent tools.

Honest caveats — Standardization is early; task coverage isn't comprehensive. Community vote doesn't equal formal standard. Adapter quality varies across agent frameworks.

What you can actually do with it — When choosing between Claude Code vs Codex vs OpenCode, check their OpenEnv scores. If evaluating agent platforms at work, ask vendors for OpenEnv results.

If any of these happen in the next week, I'll tap you again: 1) Major agent frameworks (LangChain, AutoGPT) officially adopt OpenEnv as their standard eval; 2) Karpathy or Simon Willison discuss OpenEnv; 3) Any agent tool achieves a significantly high OpenEnv score.
