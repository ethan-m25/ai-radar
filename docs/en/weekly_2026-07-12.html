🏆 The 1 thing that mattered most this week

GPT-5.6's full launch and the multi-model competitive landscape — your coding tool just got a fundamental upgrade

The core event chain this week was GPT-5.6 family moving from preview to general availability. Monday (7/6) Sol Ultra entered Codex — the tool you use every day suddenly got OpenAI's strongest flagship model. Thursday (7/9) OpenAI released the full three-tier pricing: Sol $5/$25 per million tokens, Terra $2/$10, Luna $1/$5, with a 90% cache read discount. Sam Altman called Sol "the best model we have ever produced," and Simon Willison's review showed it scoring close to Fable 5 on coding agent tasks at one-third the cost.

But simultaneously, Tencent Hy3 open-sourced (295B MoE, Apache 2.0) closely trails GLM-5.2 on multiple benchmarks (7/7), and xAI released Grok 4.5 (Cursor co-trained, $2/$6 pricing, 7/9). More competitors, lower pricing. More notably, Armin Ronacher and Simon Willison independently discovered the "Better Models: Worse Tools" phenomenon: Opus 4.8, Sonnet 5, and GPT-5.5 invent non-existent fields when calling third-party tools, causing call failures. Root cause: RL training over-optimized for specific tools (like Claude Code's built-in editor), sacrificing open-ecosystem compatibility.

If you only do one thing this week: Log into Codex and confirm Sol Ultra is available. Then create a simple test script comparing Sol against your previous model (Sonnet 5 or Opus 4) on your daily tasks — data transformation, file refactoring, shell command generation. Note output quality and response speed. Also: if you use non-official agent frameworks (Pi, Continue, custom agents), pin older model versions for now and wait for Anthropic/OpenAI to fix the degradation issue.

📈 Trends building this week

Three directions confirmed or accelerated

① Model selection economics — value-for-money replaces absolute performance as the primary decision factor

GPT-5.6's three-tier pricing (Luna $1/$5, Terra $2/$10, Sol $5/$25) plus 90% cache discount makes cost management a new skill. Compare: Opus 4.8 at $15/$75. If your daily usage is 10M input + 2M output, Sonnet 5 costs ~$40/day, Sol ~$100, Luna just $20. Meanwhile Hy3 is free on OpenRouter until July 21, and Grok 4.5 is 10x cheaper than Opus. This means you can dynamically switch models based on task complexity without being locked into a single vendor. Simon Willison's cost case study (shipping sqlite-utils 4.0rc2 with Fable 5 for $149) shows budgets can be dramatically compressed.

② Open-source models keep closing the gap — decentralized coding agent choices are growing

Tencent Hy3 (295B MoE, 21B active) closely trails GLM-5.2 on coding and agent benchmarks, and FP8 only needs ~60GB (single A100). Combined with Qwen 3.6 27B validated as the local dev sweet spot, Ornith-1.0 self-improving RL, and Leanstral 1.5 complete framework, the open-source coding model ecosystem now has "replace closed-source" potential. Alibaba's Claude Code ban (last week) and the Claude Code steganographic marking controversy (last week) further pushed enterprise interest in open-source alternatives. Copilot already supports Kimi K2.7, ZCode gave GLM-5.2 an official IDE. Open-source is no longer just a "backup" — it's becoming a mainstream option.

③ RL training side effects — newer models may regress in open ecosystems

Armin Ronacher's "Better Models: Worse Tools" revealed a structural problem: Anthropic RL-trained newer models to be better at Claude Code's built-in edit tools, but at the cost of inventing non-existent fields when calling third-party tools. Simon Willison independently confirmed the issue, and the community added that GPT-5.5 and Gemini 3.5 Flash show similar behavior. This reminds us that "upgrade = better" no longer holds. If you depend on precise tool calls (Pi, Continue, custom agents), pin older models (Sonnet 4.5 / Opus 4.5) until providers fix compatibility.
