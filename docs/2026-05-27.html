"Claude Code 当日常驱动" — HN 76 分实战长文

一篇 HN 热门文章 (76 pts, 57 comments) 拆解了 Claude Code 的全部进阶功能：Claude.md、Skills、Subagents、Plugins、MCP 生态。不卖课，不讲概念，全部是具体怎么做。你每天用 Claude Code，但你可能只用到了它能力的四分之一。这篇文章能补上剩下的四分之三。

📍 背景 · Context

Claude Code 是 Anthropic 的命令行 AI 编程工具。你已经在用它每天写代码。但它一直在快速进化：Skills 系统（自定义技能包）、Subagents（子代理工作流）、Plugins（社区插件）、MCP（模型上下文协议）集成，还有 Claude.md（项目级指令文件）。这些功能不是你打开工具就能自动发现的——它们需要主动学习和配置。这篇文章的作者 arps18 每天用 Claude Code 做主力，把他的整套配置和 workflow 公开了。

这不是新品发布，是"你买了跑车但只开过小区"型的好文章。

EN

Why this matters — this is the "power user manual" Claude Code never shipped

  • The author runs Claude Code as his primary dev environment — not a demo, not a tutorial for beginners. He structures his entire workflow around Claude.md files, custom Skills, subagent delegation, and a curated set of plugins and MCP servers. This is the level of usage clawii should be aiming for.
  • Five specific areas covered with real configs:
    1. Claude.md — per-project instruction files that tell Claude your conventions, architecture, and preferences. The author shares his actual template.
    2. Skills — reusable skill packs (e.g., a "code reviewer" skill, a "docs writer" skill) that can be loaded on demand.
    3. Subagents — spawning child Claude instances for parallel work (e.g., one subagent researches, another implements, a third reviews).
    4. Plugins — community plugins for things like Jira integration, git hooks, custom linters.
    5. MCPs — hooking Claude Code into external tools (databases, APIs, file systems) via the Model Context Protocol.
  • Why this is a signal, not just a tutorial: 57 comments on HN means the community is actively debating trade-offs — which plugins are worth it, how to structure Skills, when subagents save time vs when they add overhead. This is the "next level" conversation for daily Claude Code users. If you read this, you'll be 1-2 weeks ahead of the Chinese-language content cycle on advanced Claude Code workflows.
🎯 实操 — 你这周能做什么
  1. 花 40 分钟读原文 + HN 讨论：arps18.github.io 直接打开。重点看 Claude.md 模板和 Subagents 章节。HN 讨论在 news.ycombinator.com/item?id=42894752。
  2. 检查你的 Claude.md：你有没有给自己的项目写 Claude.md？如果没有，今天写一个。告诉 Claude 你的项目架构、命名规范、代码风格、你不想它碰的文件。作者在他的文章里贴了他的模板——直接抄。
  3. 试试 Subagents：明天挑一个中型任务（比如"整理这份薪酬数据，生成对比报告"），试试拆成"子代理 A 读数据和总结结构" → "子代理 B 写分析代码" → "主代理汇总"。对比你平时一股脑让 Claude 做的效率。记下结果。
  4. 装一个 MCP 插件：如果你用 Notion 或 Jira，先装上对应的 MCP server。然后试一个跨工具的工作流——比如"从 Notion 读取需求文档 → Claude 理解 → 写代码 → 更新 Jira ticket"。这篇文章里有具体的 MCP 配置例子。
🔬 深扒 — 这 5 个功能到底能做什么
你现在的用法（猜的）读完你可以做到的
Claude.md没有，或者只有一句 "你是一名编程助手"完整项目上下文：架构、约定、排除规则、重要文件索引
Skills不知道这个功能存在自建 "薪酬分析 skill"、"政策对比 skill"——告诉 Claude 你的领域惯例、常用公式、隐私要求
Subagents一次只做一件事，做完再问下一个同时跑多个子任务：一个查数据、一个写文案、一个检查一致性
Plugins没用过任何插件社区插件：Jira 自动更新、git 提交规范检查、安全扫描
MCPs不知道 MCP 是什么Claude 直接读数据库、操作 Notion、调用内部 API——不用你手动切换工具

坦白说：这些功能不是秘密。 Anthropic 的文档里都有。但实战配置——具体怎么写 Claude.md、怎么组织 Skills、怎么在真实项目中用 Subagents——这些文档不会教你。社区的经验才是最值钱的部分。这篇文章就是社区经验的浓缩。

对 clawii 最实用的部分：Claude.md 和 Skills。你的日常工作（薪酬分析、政策对比、数据处理）有很强的模式重复性。一个写好的 Claude.md + 几个自定义 Skills 可以让你每次开新项目时直接跳过热身阶段，Claude 一开始就懂你的行业、你的数据格式、你的合规要求。这才是真正的效率提升——不是写代码更快，而是"上手更快"。
