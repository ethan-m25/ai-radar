OpenAI Jalapeño — 首款自研推理芯片，Broadcom 代工

clawii，OpenAI 不再只靠 NVIDIA 了。它自己的芯片 Jalapeño 已经 tape out，目标是让推理成本降到「hyperscaler 级别」。这不只是新闻，未来几个月 API 价格可能会因此再跳水。

📍 背景 · 现在所有的 ChatGPT、Codex、GPT 系列模型都跑在 NVIDIA GPU 上。OpenAI 一直想减少对 NVIDIA 的依赖，同时控制成本。Jalapeño 是它第一块定制 ASIC（专用芯片），专为 LLM 推理设计，和 Broadcom 合作，设计周期只有 9 个月。社区分析认为它配备 216GB HBM3E 高带宽内存，带宽约 7.1–7.4 TB/s，FP4 算力约 10 PFLOPS。

🎯 为什么值得关注

  • 推理成本可能崩盘 — 自研芯片意味着 OpenAI 不用再付 NVIDIA 的溢价。如果推理成本下降 30-50%，你现在用的所有 AI 服务（包括 Codex、Claude 竞品）都会被逼降价。
  • 不再被 NVIDIA 卡脖子 — 芯片产能由 OpenAI 自己控制，未来模型部署更灵活，也不再受 GPU 缺货影响。
  • 对非开发者也是信号 — 你不需要管芯片细节，但未来几个月 API 价格和可用性会直接受益。
🎯 你现在能做的事：
  1. 不必立刻行动。关注 OpenAI 后续的价格公告（预计 2-3 个月后会有首批部署）。
  2. 如果你是 Costco 式囤 API 预算的人，暂时别锁定长期合约——等芯片落地后价格可能更便宜。
🔬 Deep dive — 这芯片和现在用的 GPU 有什么不同？
  • vs NVIDIA H100/B200：通用 GPU 可以训练+推理，但功耗高、成本高。Jalapeño 是纯推理 ASIC，只会做一件事（跑 LLM），所以更省电、更便宜。
  • 规格猜测（来自社区逆向）：216GB HBM3E 内存，7.1-7.4 TB/s 带宽，FP4 算力 ~10 PFLOPS。对比一下：H100 是 80GB HBM3, 3.35 TB/s, FP4 约 4 PFLOPS。单芯片推理吞吐可能是 H100 的 2-3 倍。
  • 有多大：第一代芯片规模不会太大，OpenAI 说先用于部分推理工作负载，逐步替换 GPU。
  • 为什么现在：OpenAI 在 2026 年初收购了芯片设计公司（Ona），加上 Broadcom 的制造能力，所以能这么快出片。
🔭 下一步关注的触发条件：
• OpenAI 公布 Jalapeño 实际部署后的 API 价格调整
• 类似谷歌 TPU 那样，下游模型推理降价 30%+
• 到那时 tap 你。
术语解码：
ASIC = 专用集成电路，只做一件事的芯片，比通用 GPU 省电便宜。
HBM3E = 高带宽内存第 3 代增强版，比普通内存快很多，专门喂数据给 AI 芯片。
FP4 = 4 位浮点数，用极低精度计算换取速度，是当前推理优化的主流。

Gemini 3.5 Flash — 现在可以「用」你的电脑了

clawii，Google 刚刚给 Gemini 3.5 Flash 加上了「计算机使用」能力。模型可以直接操控你的屏幕、键盘、鼠标，像人一样操作软件。这可能是 AI agent 从「聊天」到「真正替你干活」的关键一步。

📍 背景 · 你已经见过 Anthropic 的 computer use（Claude 操控电脑），但那是收费的、需要高科技栈。Google 直接把 computer use 集成到了 Gemini 3.5 Flash 里——这是一个免费可用的模型。这意味着任何开发者（包括你这样的 vibe coder）都能用自然语言让 AI 替你操作网页、填表、整理文件。

🎯 为什么值得关注

  • agent 能力大跃迁 — 不再是只输出文字，AI 能像人一样「看屏幕 → 判断 → 点击 → 反馈循环」。这对自动化重复性工作（比如入职填表、跨系统数据搬运）是革命性的。
  • 门槛低 — Gemini 3.5 Flash 在 Google AI Studio 上免费可用，不需要贵 API 或特殊硬件。
  • vibe coding 新玩法 — 你可以写一个 prompt，让 AI 自己打开浏览器、搜索、截图、写代码、提交 PR——全自动。
🎯 你现在就能做的事：
  1. 打开 Google AI Studio，选择 Gemini 3.5 Flash 模型。
  2. 尝试一个简单的 computer use 任务：帮我把这个文件夹里的简历整理成表格，然后放进 Google Sheets（需要开启 computer use 权限）。
  3. 如果效果不错，可以写一个自动化脚本每天重复。
🔬 Deep dive — 和已有的 computer use 方案有什么不同？
  • vs Anthropic Claude computer use：Claude 的 computer use 需要 Claude API 付费，且只支持 Opus 级别模型。Gemini 3.5 Flash 是免费模型，虽然能力可能稍弱，但零成本尝试。
  • vs 传统 RPA（UIPath, Automation Anywhere）：那些需要编程定义步骤，改一点就要改代码。Gemini 直接用自然语言驱动，想换就换一句话的事。
  • 技术实现：模型接收屏幕截图，生成坐标和操作指令（点击、输入、滚动）。不是通过 API 调用，而是直接操作 UI。
  • 限制：速度较慢（每次操作需要几秒推理），且对复杂界面（如拖拽、多窗口）支持有限。Google 自己说目前是「实验性」阶段。
🔭 下一步关注的触发条件：
• 更多独立评测出现，尤其是和 Claude computer use 的对比
• 如果 KOL 如 Karpathy 或 Simon Willison 单独发帖称赞，说明实用度够了，到那时 tap 你。
术语解码：
Computer use = 模型直接操控电脑屏幕、键盘、鼠标，像人一样使用软件。不是通过 API 调用，而是「看屏幕 → 操作」。
