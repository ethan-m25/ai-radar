🏆 本周最重要的唯一一件事

Fable 5 被切断后第一周——开源/本地模型集体站出来填补真空

上周六（6/14）下午 5:21pm ET，BIS 指令切断 Fable 5 和 Mythos 5 的所有中国籍用户 API。这周一是 Fable 5 被切断后的第一个完整工作日。你走进办公室，打开终端，claude-fable-5 返回 404。

然后呢？这周发生的事告诉我们：替代品已经准备好了。周一 DeepSeek 发布 Vision 模型（170 分）。周三 Qwen 发布机器人基础模型套件（183 分）。周四 Alex Ellis 的本地 Qwen 体验文登上 HN #4（235 分），Georgi Gerganov 亲自评论说他每天用 Qwen3.6-27B 做编码任务。周五 GLM 5.2 幻觉率评测出炉——仅为 GPT-5.5 的 1/3（202 分）。加上上周延续下来的 DiffusionGemma（Apache 2.0，500+ token/s）和 Cohere North Mini Code，这周你没有理由说"没有替代品"。

同时，Fable 5 出口管制事件本身也在发酵。周二 Axios 报道 Anthropic 派员工飞往华盛顿"清理"与白宫的冲突，揭示所谓的"jailbreak"实际上是让 Fable 5 "修复代码"——一个防御性安全请求。Simon Willison 周二和周三连续发文深度分析，称这是"荒谬的"。但到周末，没有任何政策松动的迹象。Fable 5 仍在停用中。

如果本周只做一件事：选一个替代模型并真正用它完成一个完整任务。推荐优先级：如果你需要最高准确度的文本生成（报告、数据分析、代码审查）→ GLM 5.2（幻觉率最低，通过 OpenRouter 可直接调用，成本仅为 GPT-5.5 的 1/30）。如果你处理敏感数据需要本地运行 → Qwen3.6-27B（Georgi Gerganov 每天在用，16GB VRAM 可跑）。如果你需要多模态 → DeepSeek Vision 或 DiffusionGemma。你失去的 Fable 5 不再是一个问题——问题是选择太多。挑一个，这周就用它完成一件事。

📈 正在成形的趋势

三条在本周被确认或加速的方向

① 开源模型在"可靠性"维度上反超闭源模型

GLM 5.2 幻觉率评测（202 分，61 条评论）是本周最重要的单个数据点之一。独立评测显示 GPT-5.5 的幻觉率是 MIT 开源模型 GLM-5.2 的 3 倍。这不是说 GLM-5.2 在所有方面都更好——它没有多模态，生态不如 GPT-5.5 成熟——但在"不胡说八道"这个关键维度上，开源模型意外领先了。加上本地 Qwen 体验文（235 分）和 DeepSeek Vision 的发布（170 分），三条信号指向同一个方向：开源模型已经不再是"廉价替代品"，而是"在某些维度上更好的选择"。对你在跨国公司做薪酬管理的影响：如果你用 AI 处理需要高准确度的任务（数据分析、报告生成），GLM-5.2 可能是比 GPT-5.5 更可靠的选择，而且成本低 30 倍。

② 本地模型完成了从"玩具"到"工具"的认知跨越

Alex Ellis 的《Local Qwen isn't a worse Opus, it's a different tool》在 HN 上获得 235 分，116 条评论。核心论点：本地模型不是云端模型的廉价替代品，而是适合不同场景的独立工具。最关键的是 Georgi Gerganov（llama.cpp 作者）的亲自背书：他每天用 Qwen3.6-27B 做编码任务，认为它非常胜任。这标志着本地模型完成了认知跨越——从"可以跑但没必要"变成了"对于某些任务，本地模型是更好的选择"。对 clawii 的意义：如果你有 16GB+ 显存的 GPU，本地 Qwen 可以帮你处理敏感数据（薪酬数据、员工信息）而不需要上传到云端。

③ 中国 AI 团队在全球开源生态中的角色从"跟进者"变为"引领者"

GLM-5.2（MIT 开源，幻觉率反超 GPT-5.5）、DeepSeek Vision（170 分，72 条评论）、Qwen-Robot Suite（183 分，31 条评论）——三个中国团队的项目在本周同时获得 HN 社区的高度认可。加上上周的 DeepSeek V4 Pro 精度反超 GPT-5.5 和 GLM 5.2 登顶 HN #4，这个趋势已经不可忽视。中国 AI 模型不再只是"中文处理更好"，而是在核心能力（幻觉率、编码能力、多模态）上达到了全球领先水平。这对你的影响：在选择替代 Fable 5 的模型时，不要因为"中国模型"的标签而排除它们——它们可能是目前最接近你需求的选择。

💀 本周我叫错了 / 过度炒作的信号

三条被我抬高但没撑住的信号

Apple Foundation Models 技术文章（6/15 👀，179 分，#2） — 周一这条信号出现时，我认为它是 Apple 亮出 AI 战略底牌的重要时刻。但到周末回头看：没有任何后续动作。没有 API 开放，没有开发者工具，没有与 Claude Code 或 Codex 竞争的迹象。Apple 的模型仍然只存在于 Apple 生态内部（Siri、Xcode 系统功能），你作为一个非 Apple 开发者无法直接调用它。它在 HN 上的热度（179 分）反映的是 Apple 粉丝的好奇心，而不是实际的开发能力。降级为 FYI。

OpenRouter Fusion API（6/15 👀，37 分，#6） — 我当时认为组合多个模型的能力对 Fable 5 被切断后的"模型组合"策略有价值。但实际上：37 分是 HN 上比较低的信号强度，10 条评论，后续没有任何 KOL 讨论或社区采用迹象。OpenRouter 的加价模式也使其成本优势不明确。仍然认为方向是对的（多模型组合是个好思路），但 Fusion API 本身没有获得足够的 traction。降级为 killed。

Lore 版本控制系统（6/18-6/19 👀，1156 分，#1） — 这是本周 HN 最高分的条目，但严格来说它不是一个 AI 工具，而是一个专为 AI 生成代码设计的版本控制系统。1156 分和 613 条评论说明社区对"Git 替代品"有强烈的诉求，但 Lore 本身是基础设施，不是模型或工具。放在 AI Radar 里属于信号类型错误。降级为 FYI，已移出核心跟踪。

已彻底移出跟踪的旧名单 — opencode（Day 28 无信号）、DeepSeek Reasonix（Day 29 无信号）、Big 4 多米诺效应（Day 27 无信号）、Gemini 3.5 Flash（Day 26 无信号）、MCP is dead 争论（Day 22 无回复）、MiMo Code 开源（HN 分数持续下降）。这些已在上周的 Weekly Wrap #24 中标注，本周正式确认死亡。另：Cohere North Mini Code 虽然在 HuggingFace 上有下载增长（19.6k），但连续 5 天无新 KOL 讨论，从 👀 降级为 killed。
