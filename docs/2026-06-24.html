Claude Tag — Slack 原生团队协作，异步委托给 Claude

hey clawii — Anthropic 在 Slack 里塞了一个完整的 Claude，你可以直接在频道里 @它分配任务。不是简单的聊天机器人，是“多角色、异步、主动”的工作流层。

📍 背景 · 你已经在用 Claude Code 做个人 coding agent，但团队协作一直是 AI 的盲区——每个成员各自对话，信息不共享，任务不同步。Claude Tag 直接在 Slack 里运行，团队可以像给人分配工作一样把任务交给 Claude，它还能跨频道协作、主动提醒。内部试用时，产品团队用它写了 65% 的代码和 PR。

🎯 Why this matters

  • 团队级 AI 工作流 — 不再是个人用 AI，而是团队共享一个 AI 同事。管理员可以控制 Claude 能访问哪些频道、工具、代码库。
  • 异步 + 主动 — Claude 可以在后台处理任务，完成后 @ 你；不像对话式 AI 需要你全程在线。
  • 对非开发者同样有用 — 你可以在 HR 频道、财务频道里让 Claude 分析数据、写报告、审核文档。
🎯 你现在就能做的事（如果你有 Claude Enterprise/Team plan）：
  1. 登录你的 Slack 工作区，在 App Directory 中安装 Claude Tag 应用。
  2. 在任意频道输入 @Claude 帮我 check 一下这份合同的条款（如果管理员已授权）
  3. 创建一个专门的 #ai-workflow 频道，把重复性任务（周报总结、数据整理）固定委托给 Claude。
🔬 Deep dive — 它和已有的 Slack AI bot 有什么不同？
  • vs Slack AI（内置）：Slack AI 只能做会话摘要和搜索；Claude Tag 可以执行复杂任务（写代码、分析数据、生成文件）并主动返回结果。
  • vs 自定义 bot（Zapier/Workato）：那些需要你预设工作流；Claude Tag 直接用自然语言在频道里“分配任务”，更像是给 Claude 发 Slack 消息。
  • 核心能力：支持读/写 Slack 消息、访问已授权的文件和代码库、后台运行异步任务。Claude 可以同时参与多个频道，互相不混淆上下文。
  • 当前只能通过 Claude Enterprise 和 Team plan 使用，beta 阶段。
🔭 下一步关注的触发条件：
如果 Claude Tag 开放给 Pro 用户，或者出现类似于「用 Claude Tag 自动处理请假审批」的企业案例，说明它正在从玩具变成基础设施。到那时 tap 你。

VibeThinker-3B — 微博 AI 的小模型逆袭，本地 Vibe Coding 新选择

hey clawii——一个只有 3B 参数的开源模型，5 天内 HuggingFace 下载量冲到 41k，star 数猛涨。它可能成为本地 vibe coding 的新主力。

📍 背景 · 你已经在用 Qwen3.6-27B 本地跑代码任务，但 27B 参数对显卡要求高（至少 16GB VRAM）。VibeThinker-3B 只有 3B 参数，目标是能在 8GB VRAM 甚至纯 CPU 上流畅运行，同时保持不错的 coding 和 agent 能力。来自微博 AI 团队（WeiboAI），不是大厂，但质量似乎不错——HF 趋势榜上冲到第 5。

🎯 Why this matters

  • 本地部署门槛大降 — 3B 参数可以用消费级显卡（6-8GB）甚至 M 芯片 Mac 的 MLX 跑。你再也不用为了跑代码助手而抢 GPU。
  • Vibe coding 专用优化 — 名字就暗示了它是为“vibe”（氛围/直觉式编程）场景设计的。从文档看，它在代码补全、错误修复、小脚本生成方面表现突出。
  • 开源 MIT — 可以随意修改、合并到自己的工作流里。
🎯 你现在就能做：
  1. 打开 Ollama（如果你还没装：curl -fsSL https://ollama.com/install.sh | sh）
  2. 拉取模型：ollama pull WeiboAI/VibeThinker-3B
  3. 在你的 vibe coding 脚本里指定这个模型（比如在 Claude Code 或 Cursor 中切换后端），测试日常任务：修复一段 Python 代码、写个脚本、改样式。
  4. 对比响应速度和输出质量，和 Qwen3.6-27B 做个记录。
🔬 Deep dive — 它和已有的小模型有什么不同？
  • vs Qwen3.6-27B：Qwen 编码能力强，但需要 16GB+ VRAM，在 M2 Ultra 上可以跑整机，但便携设备不行。VibeThinker 是 Qwen 的 1/9 参数，可以在 8GB 显卡甚至 Apple Silicon 8GB 统一内存上跑，生成速度更快。
  • vs Gemma 4 12B：Gemma 4 12B 也是小模型，但 VibeThinker 更小（3B），且专为 coding vibe 优化。Gemma 4 更偏向通用对话。
  • vs Llama 3.2 3B：Llama 3.2 3B 是 Meta 的基础模型，但 VibeThinker 在 HF 上的下载/star 增长远快于它，说明社区更认可其实际表现。
  • 技术细节：基于 LLaMA 架构，3B 参数，支持最长 32K 上下文。训练数据包含大量 Python、JavaScript、Shell 脚本。MIT 许可。
