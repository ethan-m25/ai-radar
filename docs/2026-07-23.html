POCKET：35B 参数模型可在 iPhone 上运行

爪哩，今天 HuggingFace 社区出现了一篇有趣的文章——POCKET 模型，35B 参数，却可以在 iPhone 上运行，甚至可以在没有 GPU 的 PC 上跑。如果你一直在等一个能在手机上本地运行的强大模型，这个可能是答案。

📍 背景 · 本地 AI 推理一直受限于模型大小和硬件。LM Studio、MLX、Nativ 等工具让 Mac 和 PC 上跑模型成为可能，但手机端一直缺乏大参数模型。POCKET 模型由 FINAL-Bench 团队发布，声称 35B 参数可以在 iPhone 上运行，且不需要 GPU。这是通过极致的量化和架构优化实现的吗？目前信息有限，但方向值得关注。

为什么这事重要

  • 35B 参数上手机 — 如果属实，这将是本地推理的重大突破。你可以在 iPhone 上离线运行一个接近 GPT-3.5 能力的模型，隐私和速度都更好。
  • 无需 GPU — 对于没有高端显卡的用户，POCKET 可以在普通 PC 上运行，大大降低了门槛。
  • 开源潜力 — 模型发布在 HuggingFace 上，可能是开源或开放权重的，你可以自托管或微调。

  3. 对比你当前使用的本地模型（如 LM Studio 中的模型），看 POCKET 是否真的能运行。

GigaToken：声称 1000x 语言模型 tokenization 加速

爪哩，今天 HN 上有一个项目叫 GigaToken，声称比常规 tokenization 快 1000 倍。对于经常使用 AI 编码的你来说，tokenization 是每次请求都会发生的步骤，加速意味着更快的响应和更低成本。

📍 背景 · Tokenization（分词）是将文本转换为模型可处理的 token 序列的过程。它是 LLM 推理管道的第一个步骤，通常使用 BPE 或 WordPiece 算法。GigaToken 是一个新的开源库，声称通过优化算法和并行化实现 1000 倍加速。HN 上 521 分，110 条评论，说明社区有浓厚兴趣。

为什么这事重要

  • 直接提升推理速度 — tokenization 是 LLM 推理的瓶颈之一。1000 倍加速意味着首次 token 生成时间（TTFT）大幅降低，对于实时交互场景意义重大。
  • 降低 API 成本 — 更快的 tokenization 意味着更少的计算资源消耗，可能降低使用成本。
  • 开源 — 项目在 GitHub 上，你可以集成到自己的工具链中，或者在 Codex、Claude Code 等工具中受益。
🎯 今天就能做的事
  1. 查看 GitHub 仓库：github.com/marcelroed/GigaToken
  2. 阅读 README 了解加速原理和如何使用。
  3. 如果你在本地运行模型，可以尝试替换 tokenizer 看是否兼容。

Cactus Hybrid：教 Gemma 4 知道何时出错

爪哩，HN 上有一个 Show HN 项目，叫 Cactus Hybrid，它教 Gemma 4 模型识别自己何时可能出错。虽然分数不高，但方向有趣——让模型更可靠。

📍 背景 · 模型校准（calibration）是让 AI 知道自己的不确定性。Cactus Hybrid 通过对 Gemma 4 进行微调或添加一个置信度预测头，让模型在输出时给出一个“不知道”的信号。这可以降低幻觉风险。

为什么这事重要

  • 减少幻觉 — 如果模型知道自己不确定，可以告诉你“我不确定”而不是编造答案。
  • 开源项目 — 在 GitHub 上，你可以尝试集成到自己的 agent 中。
  • 趋势信号 — 校准正在成为 agent 安全的关键组件。
