MetaGPT v0.8：让代理团队像开会一样协同

开源多代理框架 MetaGPT 今天发布了 v0.8，加入了自然语言编排（NL Orchestration）、并行子代理和动态工作流引擎。你不需要写代码就能定义一整个“代理团队”来帮你做事。

📍 背景

MetaGPT 是 GitHub 上 star 第二多的 AI 代理框架（仅次于 LangChain），主打“模拟软件公司”——你可以定义产品经理、架构师、工程师等角色，让它们协作完成开发任务。v0.8 之前，编排这些代理需要写 Python 代码或 YAML 配置文件。v0.8 的最大变化是：你用自然语言（中文/英文）描述你要做什么，MetaGPT 自己决定怎么拆任务、谁来做、怎么做。

为什么这事重要

  • 零代码编排 — 你不需要写脚本。对 clawii 这种“用 Claude Code 和 Codex 做 vibe coding”的人来说，这意味着你可以用自然语言告诉 MetaGPT“帮我写一份竞品分析报告，然后做一份 PPT，再检查一下数据一致性”，它会自己拆成子任务分给不同的代理角色。
  • 并行子代理 — 以前一个代理做完下一个才能开始。现在多个代理可以同时干活，比如一个搜资料、一个写代码、一个做图，最后合并结果。对于复杂工作流，速度翻倍。
  • 动态工作流 — 代理可以根据中间结果“改变计划”。比如写代码时发现需要先查一个 API 文档，它可以临时插入一个搜索任务再继续写。
🎯 今天就能做的事
  1. 安装 — pip install metagpt==0.8.0（需要 Python 3.11+ 和 OpenAI API key 或 Claude API key）。
  2. 试一个最简单的团队 — 运行 metagpt --run "帮我写一个网页版倒计时器，HTML+CSS+JS 在一页里"。它会自动创建 PM、架构师、工程师三个代理角色协同完成。
  3. 自定义角色 — 在 config.yaml 里加一个 role: check_quality 并说“我的角色是测试工程师，关注边缘情况”，它就会在每次输出后自动检查。
  4. 看官方 demo — 去 GitHub 仓库的 examples/ 目录看 nl_orchestration_guide.md，里面有 5 个从简单到复杂的自然语言编排示例。

跑在什么之上 — MetaGPT 支持任何 OpenAI 兼容的 API（包括 OpenAI GPT-4o/GPT-5.5、Anthropic Claude 系列、OpenRouter 上的各种模型）。v0.8 默认使用 GPT-4o（需要 API key），但你可以通过环境变量 LLM_API_KEY 和 LLM_API_BASE 切换到 Claude 或本地模型。技术栈是纯 Python 3.11+，依赖 pydantic、httpx、jinja2。

上手难度 — 安装 2 分钟（pip install）。配置 API key 30 秒。第一次运行一个简单任务大概需要 5-10 分钟，因为 MetaGPT 要启动多个角色并等待它们对话。复杂任务需要调参（比如定义角色数量、模型选择、最大 token 限制），但 v0.8 的自然语言编排让大部分场景不需要写代码。

诚实说缺点 — token 消耗大（多代理对话来回很多）；速度取决于 API 延迟（并行子代理确实快但 API 调用数翻倍）；某些任务（如写长文档）容易绕圈子；不是为实时交互设计的（它更适合跑后台任务）。

你能拿它做什么 — 让一个“分析师代理”搜资料 + “写手代理”写报告 + “质量代理”检查事实准确性，一个命令搞定。或者：让“架构师代理”设计系统 + “工程师代理”写代码 + “测试代理”跑测试 + “部署代理”推送到服务器。

Orchestration (编排)
把多个 AI 代理组织起来一起工作，就像导演指挥演员一样。不是你跟单个 AI 对话，而是你定义一群 AI 角色让它们自动协同。
Harness (马具/架子)
在 agent 工程中指“架在模型外面的那层”——包括提示词模板、工具调用、错误重试、工作流控制。MetaGPT 就是一个 harnass 框架，不是一个模型。
